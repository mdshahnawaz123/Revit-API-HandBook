Cable trays, conduits, electrical systems, panel schedules and wiring elements.

ClassCableTray
Inherits: MEPCurve → Element
Namespace: Autodesk.Revit.DB.Electrical · Assembly: RevitAPI.dll
Represents a cable tray element used to support and route electrical cables. Cable trays have width and height dimensions, a system type, and connectors at both ends for joining to fittings or other trays. Created using CableTray.Create().
🟢 Static Factory + Key Methods
🟣public static CableTray Create(Document doc, ElementId cableTrayTypeId, ElementId levelId, XYZ startPoint, XYZ endPoint)
Creates a straight cable tray segment between two 3D points at the specified level. The cable tray type determines the tray's available sizes and material. Both points must be at the same elevation for a horizontal tray.
ParameterTypeDescription
docDocumentThe target document.
cableTrayTypeIdElementIdElementId of the CableTrayType to use.
levelIdElementIdElementId of the reference level.
startPointXYZStart point of the tray in model coordinates (feet).
endPointXYZEnd point of the tray in model coordinates (feet).
Returns: CableTray — The newly created cable tray element.
// Example — Create cable tray run at ceiling height
CableTrayType ctType = new FilteredElementCollector(doc)
    .OfClass(typeof(CableTrayType))
    .Cast<CableTrayType>()
    .FirstOrDefault(t => t.Name.Contains("300"));  // 300mm wide tray

Level level = new FilteredElementCollector(doc)
    .OfClass(typeof(Level)).Cast<Level>().First();

// Create at 2700mm above level (ceiling zone)
double zFt = level.Elevation +
    UnitUtils.ConvertToInternalUnits(2700, UnitTypeId.Millimeters);

XYZ start = new XYZ(0,  0, zFt);
XYZ end   = new XYZ(20, 0, zFt);   // 6m run

using (var tx = new Transaction(doc, "Create Cable Tray"))
{
    tx.Start();
    CableTray ct = CableTray.Create(doc, ctType.Id, level.Id, start, end);

    // Set width and height
    ct.get_Parameter(BuiltInParameter.RBS_CABLETRAY_WIDTH_PARAM)
        .Set(UnitUtils.ConvertToInternalUnits(300, UnitTypeId.Millimeters));
    ct.get_Parameter(BuiltInParameter.RBS_CABLETRAY_HEIGHT_PARAM)
        .Set(UnitUtils.ConvertToInternalUnits(100, UnitTypeId.Millimeters));

    tx.Commit();
}

// Read cable tray properties
CableTray existingCT = ...;
double widthMm  = existingCT.get_Parameter(BuiltInParameter.RBS_CABLETRAY_WIDTH_PARAM)
    .AsDouble() * 304.8;
double lengthMm = (existingCT.Location as LocationCurve).Curve.Length * 304.8;

// Navigate connectors
ConnectorManager cm = existingCT.GetMEPModel().ConnectorManager;
foreach (Connector c in cm.Connectors)

var spaces = new FilteredElementCollector(doc)
    .OfCategory(BuiltInCategory.OST_MEPSpaces)
    .Cast<Space>()
    .ToList();

foreach (var space in spaces)
{
    string name     = space.Name;
    string number   = space.Number;

        BuiltInParameter.ROOM_ACTUAL_COOLING_LOAD_PARAM)
        ?.AsDouble() ?? 0;

// Create a new Space

Represents a pipe element in a plumbing or fire protection system. Pipes are round, defined by their nominal diameter, and connect to fittings and fixtures through connectors. The most critical MEP element for clash detection with structural members.
🟢 Static Factory + Properties
🟣public static Pipe Create(Document doc, ElementId systemTypeId, ElementId pipeTypeId, ElementId levelId, XYZ startPoint, XYZ endPoint)
Creates a pipe segment between two points. The PipingSystemType determines the fluid type (domestic cold water, hot water, fire sprinkler, etc.) and system classification.
// Example — Create pipe network segment
// Get domestic cold water system type
PipingSystemType coldWater = new FilteredElementCollector(doc)
    .OfClass(typeof(PipingSystemType))
    .Cast<PipingSystemType>()
    .FirstOrDefault(s => s.SystemClassification ==
        MEPSystemClassification.DomesticColdWater);

PipeType pipeType = new FilteredElementCollector(doc)
    .OfClass(typeof(PipeType)).Cast<PipeType>().First();

// Pipe height: 2500mm above level (in ceiling void)
double zFt = level.Elevation +
    UnitUtils.ConvertToInternalUnits(2500, UnitTypeId.Millimeters);

using (var tx = new Transaction(doc, "Create Pipe"))
{
    tx.Start();
    Pipe pipe = Pipe.Create(doc,
        coldWater?.Id ?? ElementId.InvalidElementId,
        pipeType.Id,
        level.Id,
        new XYZ(0, 0, zFt),
        new XYZ(16.4, 0, zFt));

    // Set diameter to 50mm
    pipe.get_Parameter(BuiltInParameter.RBS_PIPE_DIAMETER_PARAM)
        .Set(UnitUtils.ConvertToInternalUnits(50, UnitTypeId.Millimeters));

    tx.Commit();
}

// Read pipe properties
var pipes = new FilteredElementCollector(doc)
    .OfClass(typeof(Pipe)).Cast<Pipe>().ToList();

double totalRunMm = 0;
foreach (var p in pipes)
{
    double diamMm = p.get_Parameter(BuiltInParameter.RBS_PIPE_DIAMETER_PARAM)
        .AsDouble() * 304.8;
    double lenMm  = (p.Location as LocationCurve).Curve.Length * 304.8;
    totalRunMm += lenMm;

    // Check slope
    XYZ start = (p.Location as LocationCurve).Curve.GetEndPoint(0);
    XYZ end   = (p.Location as LocationCurve).Curve.GetEndPoint(1);
    double slope = Math.Abs(end.Z - start.Z) / Math.Abs(end.X - start.X + end.Y - start.Y + 0.001);
    if (slope < 0.01)
        System.Diagnostics.Debug.WriteLine($"Low slope pipe: {p.Id} slope={slope:P1}");
}

MEP Connectors

Connector, ConnectorManager — the endpoint connection system for all MEP elements.

ClassConnector
Inherits: APIObject
Namespace: Autodesk.Revit.DB
Represents a connection point on an MEP element (pipe end, duct end, fitting port, equipment port). Every MEP element has a ConnectorManager with one or more Connectors. Connectors define the size, shape, flow direction, and connection status. Two connectors on adjacent elements are connected by calling ConnectTo().
🟠 Key Properties
🟠public XYZ Origin { get; }
Gets the 3D position of this connector in model coordinates (feet). Used to find matching connectors on adjacent elements.
🟠public bool IsConnected { get; }
True if this connector is already joined to another connector. Check before calling ConnectTo() to avoid redundant connections.
🟠public ConnectorType ConnectorType { get; }
Gets the connector type: End (pipe/duct ends), Curve (along a curve), Physical (equipment port), or Logical.
🟠public double Radius { get; }
For round connectors (pipes, conduit), gets the connector radius in feet. Diameter = Radius × 2.
🟠public ConnectorSet AllRefs { get; }
Gets all connectors that this connector is connected to. Typically one for pipe ends (the matching connector on the next element).
🟢 Key Methods
🟢public void ConnectTo(Connector connector)
Connects this connector to another connector. Both connectors must be unconnected, compatible (same size/shape), and in close proximity. Must be called inside a Transaction.
// Example — Connect two pipe endpoints
// Helper: find connector nearest to a 3D point
Connector GetConnectorAt(Element elem, XYZ point, double tol = 0.01)
{
    ConnectorManager cm = (elem as MEPCurve)?.GetMEPModel()?.ConnectorManager
                        ?? (elem as FamilyInstance)?.GetMEPModel()?.ConnectorManager;
    if (cm == null) return null;
    foreach (Connector c in cm.Connectors)
        if (c.Origin.DistanceTo(point) < tol) return c;
    return null;
}

// Connect pipe1 end to pipe2 start
Pipe pipe1 = ...;
Pipe pipe2 = ...;

XYZ pipe1End   = (pipe1.Location as LocationCurve).Curve.GetEndPoint(1);
XYZ pipe2Start = (pipe2.Location as LocationCurve).Curve.GetEndPoint(0);

using (var tx = new Transaction(doc, "Connect Pipes"))
{
    tx.Start();
    Connector c1 = GetConnectorAt(pipe1, pipe1End);
    Connector c2 = GetConnectorAt(pipe2, pipe2Start);

    if (c1 != null && c2 != null && !c1.IsConnected && !c2.IsConnected)
        c1.ConnectTo(c2);

    tx.Commit();
}

// Traverse a connected network from a seed element
Pipe seedPipe = ...;
var visited  = new HashSet<ElementId>();
var queue    = new Queue<Element>();
queue.Enqueue(seedPipe);

while (queue.Count > 0)
{
    Element cur = queue.Dequeue();
    if (!visited.Add(cur.Id)) continue;

    ConnectorManager cm = (cur as MEPCurve)?.GetMEPModel()?.ConnectorManager;
    if (cm == null) continue;

    foreach (Connector c in cm.Connectors)
        foreach (Connector neighbor in c.AllRefs)
        {
            Element next = doc.GetElement(neighbor.Owner.Id);
            if (next != null && !visited.Contains(next.Id))
                queue.Enqueue(next);
        }
}

ClassRoom
Inherits: SpatialElement → Element
Namespace: Autodesk.Revit.DB.Architecture
Represents an architectural room — a bounded space with name, number, area, volume, and finish data. Rooms are placed in floor plan views and bounded by walls, separation lines, or room bounding elements. The Room is the most important element for space management, area calculations, and BIM data assignment.
🟠 Key Properties
🟠public string Name { get; set; }
Gets or sets the room name. Displayed in the room tag. Setting requires an open Transaction.
🟠public string Number { get; set; }
Gets or sets the room number. Unique identifier for the room within the project.
🟠public double Area { get; }
Gets the floor area of the room in sq.ft internal units. Multiply by 0.092903 for m². Returns 0 for unplaced rooms.
🟠public double Volume { get; }
Gets the room volume in cubic feet. Only computed when Room Volume calculation is enabled in project settings.
🟠public Level Level { get; }
Gets the Level this room is placed on.
🟢 Key Methods
🟢public bool IsPointInRoom(XYZ point)
Returns true if the given 3D point is inside this room's boundary. Used for spatial queries: finding which room a pipe passes through, assigning rooms to equipment, etc. The point must be within the room's elevation range.
ParameterTypeDescription
pointXYZThe point to test (in model coordinates, feet).
Returns: bool — true if the point is inside this room.
🟢public IList<IList<BoundarySegment>> GetBoundarySegments(SpatialElementBoundaryOptions options)
Returns the boundary of the room as a list of loops, each containing BoundarySegment objects. Index 0 is the outer boundary; subsequent loops are holes. Each BoundarySegment identifies the bounding wall element.
// Example — Complete Room operations
// Collect all placed rooms (Area > 0)
var rooms = new FilteredElementCollector(doc)
    .OfCategory(BuiltInCategory.OST_Rooms)
    .Cast<Room>()
    .Where(r => r.Area > 1)    // placed rooms only
    .OrderBy(r => r.Number)
    .ToList();

foreach (var room in rooms)
{
    string name    = room.Name;
    string number  = room.Number;
    double areaSqM = room.Area * 0.092903;
    string dept    = room.get_Parameter(BuiltInParameter.ROOM_DEPARTMENT)
                        ?.AsString() ?? "";

    // Get room boundary to create a floor
    var bOpts = new SpatialElementBoundaryOptions();
    IList<IList<BoundarySegment>> segs = room.GetBoundarySegments(bOpts);

    if (segs.Count > 0)
    {
        CurveLoop boundary = new CurveLoop();
        foreach (var seg in segs[0])  // outer loop only
            boundary.Append(seg.GetCurve());

        // Identify bounding walls
        foreach (var seg in segs[0])
        {
            ElementId wallId = seg.ElementId;
            if (wallId != ElementId.InvalidElementId)
            {
                Wall w = doc.GetElement(wallId) as Wall;
                System.Diagnostics.Debug.WriteLine(
                    $"  Bounded by: {w?.WallType?.Name}");
            }
        }
    }

    // Check if a point is in the room
    XYZ testPt = new XYZ(5, 5, room.Level.Elevation + 1);
    bool inside = room.IsPointInRoom(testPt);
}

// Create a room programmatically
using (var tx = new Transaction(doc, "Create Room"))
{
    tx.Start();
    UV   location = new UV(5, 5);  // XY position on level
    Room newRoom  = doc.Create.NewRoom(level, location);
    newRoom.Name   = "Conference Room";
    newRoom.Number = "101";
    tx.Commit();
}
ClassTopographySurface
Inherits: Element
Namespace: Autodesk.Revit.DB.Architecture
Represents the topography/site surface. Contains a mesh of 3D points defining the terrain. Points can be read, added, moved, or deleted. Sub-regions are child TopographySurface elements with different materials applied to portions of the surface.
🟢 Key Methods
🟢public IList<XYZ> GetPoints()
Returns all the XYZ points that define the topography mesh. Points are in model coordinates (feet). Useful for elevation analysis, contour generation, and cut/fill calculations.
Returns: IList<XYZ> — All mesh points defining the terrain surface.
// Example — Topography analysis and export
TopographySurface topo = new FilteredElementCollector(doc)
    .OfClass(typeof(TopographySurface))
    .Cast<TopographySurface>()
    .FirstOrDefault(t => !t.IsSiteSubRegion);

if (topo == null) return Result.Failed;

IList<XYZ> pts = topo.GetPoints();

// Statistics
double minElev = pts.Min(p => p.Z) * 304.8;
double maxElev = pts.Max(p => p.Z) * 304.8;
double avgElev = pts.Average(p => p.Z) * 304.8;

TaskDialog.Show("Topo",
    $"Points: {pts.Count}\nMin: {minElev:F0}mm\nMax: {maxElev:F0}mm\nAvg: {avgElev:F0}mm");

// Export as XYZ CSV
var csv = new System.Text.StringBuilder("X,Y,Z\n");
foreach (var p in pts)
    csv.AppendLine($"{p.X * 304.8:F3},{p.Y * 304.8:F3},{p.Z * 304.8:F3}");

// Modify topo points
using (TopographyEditScope scope = new TopographyEditScope(doc, "Edit Topo"))
{
    scope.Start(topo.Id);
    using (var tx = new Transaction(doc, "Move Points"))
    {
        tx.Start();
        // Raise points within a region by 500mm
        var ptsToRaise = pts.Where(p => p.X > 0 && p.Y > 0).ToList();
        double raise = UnitUtils.ConvertToInternalUnits(500, UnitTypeId.Millimeters);
        var newPts = ptsToRaise.Select(p => new XYZ(p.X, p.Y, p.Z + raise)).ToList();
        topo.ChangePointsToElevation(ptsToRaise, newPts);
        tx.Commit();
    }
    scope.Commit(new TopographyEditFailuresPreprocessor());
}

// Example — Read stair properties and check compliance

foreach (var stair in stairs)

    int    risers   = stair.get_Parameter(
        BuiltInParameter.STAIRS_RISER_NUMBER)?.AsInteger() ?? 0;
    double riserH   = stair.get_Parameter(
        BuiltInParameter.STAIRS_ACTUAL_RISER_HEIGHT)?.AsDouble() * 304.8 ?? 0;
    double treadD   = stair.get_Parameter(
        BuiltInParameter.STAIRS_ACTUAL_TREAD_DEPTH)?.AsDouble() * 304.8 ?? 0;

    // UK Building Regs: max riser 220mm, min going 220mm
    bool riserOK  = riserH <= 220;
    bool treadOK  = treadD >= 220;

    System.Diagnostics.Debug.WriteLine(
        $"Stair {stair.Id}: Risers={risers} H={riserH:F0}mm T={treadD:F0}mm " +
        $"{(riserOK && treadOK ? "PASS" : "FAIL")}");
}

// Dynamic Model Update

IUpdater & UpdaterRegistry

The DMU framework lets you react to model changes by modifying the model — unlike DocumentChanged which is read-only.

InterfaceIUpdater
Namespace: Autodesk.Revit.DB
The core interface for Dynamic Model Update. Implement this to react to element changes with model writes. Unlike DocumentChanged (read-only), DMU updaters fire INSIDE the triggering transaction, so you can modify related elements in response to changes without starting a new transaction.
// Warning — No New TransactionsInside IUpdater.Execute(), you are already inside an open transaction. Do NOT start a new Transaction — just modify elements directly. Starting a nested transaction will throw InvalidOperationException.
🟢 Required Methods + Complete Pattern
// Example — Complete IUpdater implementation
public class WallMarkAutoUpdater : IUpdater
{
    private static AddInId   _appId;
    private static UpdaterId _id;

    public WallMarkAutoUpdater(AddInId appId)
    {
        _appId = appId;
        _id    = new UpdaterId(appId,
            new Guid("AABBCCDD-1234-5678-ABCD-EF0123456789"));
    }

    // Fires INSIDE the active transaction — no new Transactions here!
    public void Execute(UpdaterData data)
    {
        Document doc = data.GetDocument();

        // Handle modified walls
        foreach (ElementId id in data.GetModifiedElementIds())
        {
            Wall wall = doc.GetElement(id) as Wall;
            if (wall == null) continue;

            // Auto-update Mark with length
            double lenMm = (wall.Location as LocationCurve)
                .Curve.Length * 304.8;
            Parameter mark = wall.get_Parameter(
                BuiltInParameter.ALL_MODEL_MARK);
            if (mark != null && !mark.IsReadOnly)
                mark.Set($"W-{lenMm:F0}mm");  // write directly, no tx needed
        }

        // Handle newly added walls
        foreach (ElementId id in data.GetAddedElementIds())
        {
            Wall wall = doc.GetElement(id) as Wall;
            if (wall == null) continue;
            // Tag new walls automatically
            wall.get_Parameter(BuiltInParameter.ALL_MODEL_MARK)
                ?.Set($"NEW-{wall.Id.IntegerValue}");
        }
    }

    public string      GetUpdaterName()             => "Wall Mark Auto Updater";
    public string      GetAdditionalInformation()   => "Auto-tags walls with length on change";
    public ChangePriority GetChangePriority()       => ChangePriority.MEPFixtures;
    public UpdaterId  GetUpdaterId()               => _id;
}

// ── Register in OnStartup ──
public Result OnStartup(UIControlledApplication app)
{
    var updater = new WallMarkAutoUpdater(app.ActiveAddInId);
    UpdaterRegistry.RegisterUpdater(updater);

    // Trigger: fire on geometry change of any Wall
    UpdaterRegistry.AddTrigger(
        updater.GetUpdaterId(),
        new ElementClassFilter(typeof(Wall)),
        Element.GetChangeTypeGeometry());

    // Also trigger on any Wall addition
    UpdaterRegistry.AddTrigger(
        updater.GetUpdaterId(),
        new ElementClassFilter(typeof(Wall)),
        Element.GetChangeTypeElementAddition());

    return Result.Succeeded;
}

// ── Unregister in OnShutdown ──
public Result OnShutdown(UIControlledApplication app)
{
    UpdaterRegistry.UnregisterUpdater(_updater.GetUpdaterId());
    return Result.Succeeded;
}
ClassUpdaterRegistry
Inherits: Static utility
Namespace: Autodesk.Revit.DB
Static class for registering, unregistering, and querying IUpdater instances. All DMU updaters must be registered before they can receive triggers. Updaters are registered per session — they must be re-registered on every Revit startup.
🟢 Key Static Methods
🟢public static void RegisterUpdater(IUpdater updater)
Registers an IUpdater so it can receive trigger events. Call in OnStartup(). Does NOT add any triggers — call AddTrigger() separately.
