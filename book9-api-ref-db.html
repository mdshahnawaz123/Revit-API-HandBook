Class AddInId
Inherits: System.Object
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents the unique identifier for a Revit add-in. Used to register updaters (DMU), identify the add-in in event handlers, and create UpdaterId instances. Every IExternalApplication and IExternalCommand has an associated AddInId provided by Revit at runtime.
🟣 Constructors
🟣 public AddInId(Guid guid)
Creates an AddInId from a GUID. This GUID must match the GUID specified in the add-in's .addin manifest file exactly. Mismatches will cause registration failures at runtime.
ParameterTypeDescription
guidGuidThe unique GUID that identifies this add-in. Must match the addin manifest GUID.
// Example — AddInId Constructor
// In IExternalApplication.OnStartup — Revit provides the AddInId
public Result OnStartup(UIControlledApplication app)
{
    // Revit gives you the AddInId automatically
    AddInId addInId = app.ActiveAddInId;
    Guid guid = addInId.GetGUID();

    // Or create one manually (must match .addin file GUID)
    Guid myGuid = new Guid("12345678-1234-1234-1234-123456789abc");
    AddInId manualId = new AddInId(myGuid);

    // Used when creating an UpdaterId for DMU
    UpdaterId updaterId = new UpdaterId(addInId,
        new Guid("aaaabbbb-cccc-dddd-eeee-ffffaaaabbbb"));

    return Result.Succeeded;
}
🟢 Methods
🟢 public Guid GetGUID()
Returns the GUID that uniquely identifies this add-in. This GUID corresponds to the <AddInId> value in the .addin manifest XML file.

Class Arc
Inherits: Curve → APIObject
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents a circular arc in 3D space. Arcs are defined by a center, radius, start angle and end angle. As a subclass of Curve, Arc inherits all curve operations including parameter evaluation, projection, tessellation, and endpoint access. Arcs are used for curved walls, circular dimensions, and any circular geometry in the model.
// Note Arc is immutable — all Create() methods return a new Arc instance. You cannot modify a created arc in-place. Use the appropriate Create() overload for your use case: three points, center+radius+angles, or plane+radius+angles.
🟣 Static Factory Methods (Creation)
🟣 public static Arc Create(XYZ end0, XYZ end1, XYZ pointOnCurve)
Creates a new arc passing through three points. The arc passes through end0, pointOnCurve, and end1. This is the most common overload when you know three points on the arc but not the center.
ParameterTypeDescription
end0XYZThe start point (first endpoint) of the arc.
end1XYZThe end point (second endpoint) of the arc.
pointOnCurveXYZA point that lies on the arc between the two endpoints. Cannot be collinear with end0 and end1.
Returns: Arc — A new Arc object passing through all three points.
// Example — Arc.Create(3 points)
// Create an arc from three known points
XYZ start  = new XYZ(0, 0, 0);
XYZ mid    = new XYZ(5, 5, 0);    // point on arc
XYZ end    = new XYZ(10, 0, 0);

Arc arc = Arc.Create(start, end, mid);

// Read arc properties
XYZ    center = arc.Center;                // center point
double radius = arc.Radius;               // radius in feet
double startA = arc.GetEndParameter(0);  // start angle (radians)
double endA   = arc.GetEndParameter(1);  // end angle (radians)
bool   isBound = arc.IsBound;             // true for bounded arc

TaskDialog.Show("Arc",
    $"Center: {center}\nRadius: {radius * 304.8:F1}mm");
🟣 public static Arc Create(Plane plane, double radius, double startAngle, double endAngle)
Creates a new arc on a specified plane with a given radius, start angle and end angle. The center of the arc is the origin of the plane. Angles are measured from the plane's X-axis in the plane's XY plane.
ParameterTypeDescription
planePlaneThe plane that contains the arc. The origin is the arc center.
radiusdoubleThe radius of the arc in internal units (feet).
startAngledoubleThe start angle of the arc in radians, measured from plane's X-axis.
endAngledoubleThe end angle of the arc in radians. Must be greater than startAngle.
Returns: Arc — A new bounded Arc on the specified plane.
// Example — Arc.Create(Plane, radius, angles) — Quarter circle
// Create a quarter-circle arc (0° to 90°) at radius 5000mm
Plane xyPlane = Plane.CreateByNormalAndOrigin(
    XYZ.BasisZ,
    new XYZ(0, 0, 0));

double radiusFt   = UnitUtils.ConvertToInternalUnits(5000, UnitTypeId.Millimeters);
double startAngle = 0;                   // 0 degrees
double endAngle   = Math.PI / 2;        // 90 degrees

Arc quarterArc = Arc.Create(xyPlane, radiusFt, startAngle, endAngle);

// Use to create a curved wall
using (var tx = new Transaction(doc, "Curved Wall"))
{
    tx.Start();
    Wall.Create(doc, quarterArc, wallType.Id, level.Id,
        3.0, 0.0, false, false);
    tx.Commit();
}
🟣 public static Arc Create(XYZ center, double radius, double startAngle, double endAngle, XYZ xAxis, XYZ yAxis)
Creates a new arc with explicit center, radius, angles, and axis vectors. This overload gives full control over the arc's orientation in 3D space without requiring a Plane object. xAxis and yAxis define the arc's reference frame.
ParameterTypeDescription
centerXYZThe center point of the arc's full circle.
radiusdoubleThe radius in internal units (feet).
startAngledoubleStart angle in radians from xAxis.
endAngledoubleEnd angle in radians from xAxis.
xAxisXYZUnit vector defining 0° direction (must be unit length).
yAxisXYZUnit vector perpendicular to xAxis in the arc plane.
Returns: Arc — A new Arc with the specified geometry.
// Example — Arc.Create(center, radius, angles, axes)
// Create a tilted arc in the XZ plane (vertical arc)
XYZ center = new XYZ(5, 0, 3);
double radius = 2.0;          // 2 feet
XYZ xAxis  = XYZ.BasisX;     // horizontal
XYZ yAxis  = XYZ.BasisZ;     // vertical — makes this a vertical arc

Arc verticalArc = Arc.Create(
    center, radius,
    0,              // start: 0°
    Math.PI,        // end: 180° (semicircle)
    xAxis, yAxis);

// Tessellate the arc to get polyline approximation
IList<XYZ> pts = verticalArc.Tessellate();
foreach (var pt in pts)
    System.Diagnostics.Debug.WriteLine($"Point: ({pt.X:F3}, {pt.Y:F3}, {pt.Z:F3})");

🟢public Transform ComputeDerivatives(double parameter, bool normalized)
Returns the position and first/second derivative vectors of the arc at the given parameter. The Transform's origin is the point on the curve, BasisX is the tangent vector, BasisY is the normal (pointing toward center), and BasisZ is the binormal.
Returns: Transform — A frame where Origin=point, BasisX=tangent, BasisY=normal, BasisZ=binormal.
// Example — Tangent direction at arc midpoint
Arc       arc        = ...;
Transform derivs     = arc.ComputeDerivatives(0.5, true);

XYZ pointOnArc = derivs.Origin;   // position at parameter 0.5
XYZ tangent    = derivs.BasisX;   // tangent direction (normalized)
XYZ normal     = derivs.BasisY;   // points toward center

// Place element aligned to arc tangent at midpoint
Transform placement = Transform.Identity;
placement.Origin  = pointOnArc;
placement.BasisX  = tangent;
placement.BasisY  = XYZ.BasisZ;
placement.BasisZ  = tangent.CrossProduct(XYZ.BasisZ);
Class BoundingBoxXYZ
Inherits: APIObject
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents an axis-aligned bounding box defined by minimum and maximum XYZ points, plus an optional Transform. Used extensively for: getting element extents, creating section views (BoundingBoxXYZ defines the section box), defining 3D view section boxes, and spatial queries. The Transform defines a local coordinate frame — Min and Max are in the transform's local space.
// Critical — Section Views When creating section views with ViewSection.CreateSection(), the BoundingBoxXYZ MUST have its Transform set. A BoundingBoxXYZ with Transform = Identity will create a section at the world origin. Set Transform.Origin to the cut location and Transform.BasisX/Y/Z to the view orientation.
🟣 Constructors
🟣public BoundingBoxXYZ()
Creates a new empty BoundingBoxXYZ with Min and Max set to invalid values and Transform set to Identity. You must set Min, Max, and typically Transform before use.
// Example — Create BoundingBoxXYZ for a section view
// Creating a section perpendicular to a wall
Wall          wall    = ...;
LocationCurve lc      = wall.Location as LocationCurve;
XYZ           midPt   = lc.Curve.Evaluate(0.5, true);
XYZ           wallDir = (lc.Curve.GetEndPoint(1) - lc.Curve.GetEndPoint(0)).Normalize();
XYZ           viewDir = wallDir.CrossProduct(XYZ.BasisZ).Normalize();

// Build the Transform (defines section orientation)
Transform t = Transform.Identity;
t.Origin  = midPt;
t.BasisX  = wallDir;     // horizontal in view
t.BasisY  = XYZ.BasisZ; // vertical in view
t.BasisZ  = viewDir;     // camera direction

// Build the bounding box (extents in local space)
BoundingBoxXYZ bb = new BoundingBoxXYZ();
bb.Transform = t;
bb.Min = new XYZ(-5, 0, -1);    // left, bottom, near (feet)
bb.Max = new XYZ( 5, 10, 1);   // right, top, far (feet)

// Create the section view
ElementId sectionTypeId = new FilteredElementCollector(doc)
    .OfClass(typeof(ViewFamilyType))
    .Cast<ViewFamilyType>()
    .First(x => x.ViewFamily == ViewFamily.Section).Id;

using (var tx = new Transaction(doc, "Section"))
{
    tx.Start();
    ViewSection sec = ViewSection.CreateSection(doc, sectionTypeId, bb);
    sec.Name = "Wall Section";
    tx.Commit();
}
🟠 Properties
🟠public XYZ Min { get; set; }
Gets or sets the minimum corner of the bounding box in the Transform's local coordinate space. For a standard element bounding box, this is the bottom-left-near corner.
🟠public XYZ Max { get; set; }
Gets or sets the maximum corner of the bounding box in local space. For an element bounding box this is the top-right-far corner.
🟠public Transform Transform { get; set; }
Gets or sets the transform that maps from the bounding box's local coordinate space to the world (model) coordinate space. For element bounding boxes this is often Identity. For section views this defines the cut orientation.
🟠public bool Enabled { get; set; }
Gets or sets whether the bounding box is enabled. Used primarily with View3D.SetSectionBox — setting Enabled = false on the section box disables section box clipping in the 3D view.
// Example — Element bounding box and centroid
Element elem = ...;

// Get element bounding box (pass null for model extents, view for view-clipped)
BoundingBoxXYZ bb = elem.get_BoundingBox(null);

if (bb != null)
{
    XYZ min = bb.Min;
    XYZ max = bb.Max;

    // Compute centroid
    XYZ center = new XYZ(
        (min.X + max.X) / 2,
        (min.Y + max.Y) / 2,
        (min.Z + max.Z) / 2);

    // Dimensions in mm
    double widthMM  = (max.X - min.X) * 304.8;
    double depthMM  = (max.Y - min.Y) * 304.8;
    double heightMM = (max.Z - min.Z) * 304.8;

    TaskDialog.Show("BBox",
        $"W:{widthMM:F0} D:{depthMM:F0} H:{heightMM:F0}mm");
}

Class Color

Class CurveLoop
Inherits: APIObject · IEnumerable<Curve>
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents a closed or open sequence of connected curves. CurveLoop is the primary way to define profiles for Floor.Create(), ceiling profiles, void extrusions, filled regions, and room separators. The curves must be connected end-to-start — the endpoint of each curve must match the start of the next within tolerance.
// Warning — Curve Connectivity All curves in a CurveLoop must be end-to-end connected. If curves don't connect within Revit's tolerance, API calls using the CurveLoop will fail with an exception. Use CurveLoop.IsOpen() to verify, and ensure curves are appended in order.
🟣 Constructors
🟣public CurveLoop()
Creates an empty CurveLoop. Curves are added using Append(). The loop can also be created from an existing IEnumerable of curves using CreateViaThicken() or static factory methods.
// Example — CurveLoop for a rectangular floor
// Create a 6m × 4m rectangular room profile
XYZ p0 = new XYZ(0,  0, 0);
XYZ p1 = new XYZ(20, 0, 0);   // 6096mm ≈ 20 feet
XYZ p2 = new XYZ(20, 13, 0);  // ~4m
XYZ p3 = new XYZ(0,  13, 0);

CurveLoop loop = new CurveLoop();
loop.Append(Line.CreateBound(p0, p1));
loop.Append(Line.CreateBound(p1, p2));
loop.Append(Line.CreateBound(p2, p3));
loop.Append(Line.CreateBound(p3, p0));

// Verify it's closed
bool isOpen = loop.IsOpen();   // should be false
double perimeter = loop.GetExactLength();

// Use to create a floor
FloorType ft = new FilteredElementCollector(doc)
    .OfClass(typeof(FloorType)).Cast<FloorType>().First();
Level lv = new FilteredElementCollector(doc)
    .OfClass(typeof(Level)).Cast<Level>().First();

using (var tx = new Transaction(doc, "Create Floor"))
{
    tx.Start();
    Floor.Create(doc, new List<CurveLoop> { loop }, ft.Id, lv.Id);
    tx.Commit();
}

Inherits: APIObject · All model elements inherit from this class

The base class for every element in the Revit model. Both instance elements (Wall, Floor, FamilyInstance) and type elements (WallType, FamilySymbol) derive from Element. Provides access to universal properties: Id, Name, Category, Parameters, Geometry, BoundingBox, Level, and dependent elements.

🟠public string Name { get; set; }
Gets or sets the element's name. For instances this is typically empty; for types (WallType, FamilySymbol) this is the type name. Setting Name requires an open Transaction.
🟠public Category Category { get; }
Gets the Revit category this element belongs to. The Category.Id.IntegerValue can be cast to BuiltInCategory for comparison. May be null for some element types.
🟠public string UniqueId { get; }
Gets the globally unique identifier (GUID string) for this element. Unlike Id (integer), UniqueId is stable across documents and is used for IFC GUID mapping and cross-document element tracking.
🟠public ParameterSet Parameters { get; }
Gets the complete set of all parameters available on this element (both instance and type parameters accessible through the instance). Iterate to discover all parameters or use get_Parameter() for direct access.
🟠public ElementId LevelId { get; }

🟢public GeometryElement get_Geometry(Options options)
Returns the geometric representation of the element. The Options object controls: which view controls visibility, whether references are computed (required for dimensioning), and whether invisible objects are included.
// Example — Extracting geometry from an element

// Configure options
var opts = new Options()
{
    ComputeReferences       = true,    // REQUIRED for dimensioning
    IncludeNonVisibleObjects = false,
    View                    = doc.ActiveView
};

GeometryElement geomElem = elem.get_Geometry(opts);
double totalVolume = 0;

foreach (GeometryObject obj in geomElem)
{
    if (obj is Solid solid && solid.Volume > 1e-9)
    {
        totalVolume += solid.Volume;

        foreach (Face face in solid.Faces)
        {
            Reference faceRef = face.Reference;   // for dimensions
            XYZ normal = face.ComputeNormal(new UV(.5, .5));
        }
    }
    else if (obj is GeometryInstance gi)
    {
        // FamilyInstance geometry is inside GeometryInstance
        foreach (GeometryObject instObj in gi.GetInstanceGeometry())
        {
            if (instObj is Solid s && s.Volume > 1e-9)
                totalVolume += s.Volume;
        }
    }
}
TaskDialog.Show("Volume",
    $"{totalVolume * 0.0283168:F4} m³");

Class FilteredElementCollector
Inherits: APIObject · IEnumerable<Element> · IEnumerable<ElementId>
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
The primary query engine for retrieving elements from a Revit document. Supports both quick (native) filters that run in Revit's engine with near-zero overhead, and slow (LINQ) filters that iterate element by element. Always apply quick filters first, then LINQ filters. The collector is lazy — elements are retrieved when the result is enumerated.
// Performance Tip Chain quick filters in this order for maximum performance: OfClass() → OfCategory() → WhereElementIsNotElementType() → WherePasses(nativeFilter). Apply LINQ .Where() only after all native filters. Use .ToElementIds() when you only need IDs — it avoids loading full element objects.
🟣 Constructors
🟣public FilteredElementCollector(Document document)
Creates a collector that will search the entire document. This is the most common constructor — it searches all elements in all views and all levels.
🟣public FilteredElementCollector(Document document, ElementId viewId)
Creates a collector restricted to elements visible in the specified view. Only elements that are visible (not hidden, not in hidden categories) in the view will be returned. This respects crop regions, workset visibility, and view-specific overrides.
// Example — FilteredElementCollector patterns
// ── QUICK FILTERS (native, fast) ──

// By class (fastest)
var walls = new FilteredElementCollector(doc)
    .OfClass(typeof(Wall))
    .Cast<Wall>()
    .ToList();

// By category — instances only
var doors = new FilteredElementCollector(doc)
    .OfCategory(BuiltInCategory.OST_Doors)
    .WhereElementIsNotElementType()
    .ToList();

// Types only
var wallTypes = new FilteredElementCollector(doc)
    .OfClass(typeof(WallType))
    .Cast<WallType>()
    .ToList();

// In a specific view only
var visibleWalls = new FilteredElementCollector(doc, doc.ActiveView.Id)
    .OfClass(typeof(Wall))
    .Cast<Wall>()
    .ToList();

// ── PARAMETER FILTER (native, slower than OfClass but still native) ──
var markProvider = new ParameterValueProvider(
    new ElementId(BuiltInParameter.ALL_MODEL_MARK));
var rule = new FilterStringRule(
    markProvider, new FilterStringContains(), "EXT");
var extWalls = new FilteredElementCollector(doc)
    .OfClass(typeof(Wall))
    .WherePasses(new ElementParameterFilter(rule))
    .Cast<Wall>()
    .ToList();

// ── MULTIPLE CATEGORIES ──
var multiFilter = new ElementMulticategoryFilter(
    new List<BuiltInCategory>
    { BuiltInCategory.OST_Walls, BuiltInCategory.OST_Floors });
var structural = new FilteredElementCollector(doc)
    .WherePasses(multiFilter)
    .WhereElementIsNotElementType()
    .ToList();

// ── ELEMENT IDs ONLY (fast, less memory) ──
ICollection<ElementId> ids = new FilteredElementCollector(doc)
    .OfClass(typeof(Wall))
    .ToElementIds();

// ── LINQ FILTER (after native filters) ──
var longWalls = new FilteredElementCollector(doc)
    .OfClass(typeof(Wall))
    .Cast<Wall>()
    .Where(w => (w.Location as LocationCurve)?.Curve.Length > 10)
    .ToList();
🟢 Quick Filter Methods
🟢public FilteredElementCollector OfClass(Type type)
Restricts the collector to elements of the specified .NET class type. This is the fastest filter available — runs entirely in Revit's native engine. Note: OfClass(typeof(View)) includes all view subtypes; use LINQ to further refine.
🟢public FilteredElementCollector OfCategory(BuiltInCategory category)
Restricts the collector to elements belonging to the specified built-in category. By default includes both instances and types — chain with WhereElementIsNotElementType() to get instances only.
🟢public FilteredElementCollector WhereElementIsNotElementType()
Restricts results to instance elements only (excludes WallType, FamilySymbol, etc.). Pair with OfCategory() to get placed instances.
🟢public FilteredElementCollector WhereElementIsElementType()
Restricts results to type elements only (WallType, FloorType, FamilySymbol, etc.). Use this to collect all types of a category.
🟢public FilteredElementCollector WherePasses(ElementFilter filter)
Applies any ElementFilter to the collector. Use with: ElementParameterFilter, ElementLevelFilter, BoundingBoxIntersectsFilter, ElementMulticategoryFilter, LogicalAndFilter, LogicalOrFilter, and ExclusionFilter.
🟢 Retrieval Methods
🟢public IList<Element> ToElements()
Executes the query and returns all matching elements as a list. The most common retrieval method. Returns an empty list (not null) if no elements match.
🟢public ICollection<ElementId> ToElementIds()
Executes the query and returns element IDs only. Significantly faster and less memory-intensive than ToElements() when you only need IDs for deletion, selection, or external storage.
🟢public Element FirstElement()
Returns the first matching element, or null if no elements match. Use when you expect exactly one result (e.g. finding a specific level by name). More efficient than ToElements().FirstOrDefault().
