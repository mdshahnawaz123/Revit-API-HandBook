// Autodesk.Revit.UI — Ribbon

Ribbon API

Create fully custom ribbon tabs, panels, and all 8 control types — PushButton, SplitButton, PulldownButton, ComboBox, TextBox, RadioButtonGroup, Slider, and Stacked Items.

// Ribbon Hierarchy UIControlledApplication → RibbonTab → RibbonPanel → RibbonItem (PushButton, SplitButton, etc.). All ribbon construction must happen in IExternalApplication.OnStartup(). You cannot add ribbon items after startup — only enable/disable/update them at runtime.
UIControlledApplication ├── CreateRibbonTab("Tab Name") → custom tab ├── CreateRibbonPanel("Panel Name") → panel on Add-Ins tab └── CreateRibbonPanel("Tab", "Panel") → panel on specific tab RibbonPanel ├── AddItem(PushButtonData) → PushButton ├── AddItem(SplitButtonData) → SplitButton ├── AddItem(PulldownButtonData) → PulldownButton ├── AddItem(ComboBoxData) → ComboBox ├── AddItem(TextBoxData) → TextBox ├── AddItem(RadioButtonGroupData) → RadioButtonGroup ├── AddItem(SliderData) → Slider └── AddStackedItems(data1, data2, data3) → 2 or 3 stacked small buttons
ClassUIControlledApplication
Inherits: APIObject
Namespace: Autodesk.Revit.UI · Available only in IExternalApplication.OnStartup()
The entry point for all ribbon construction. Provided as the parameter to IExternalApplication.OnStartup(). Used to create ribbon tabs, panels, register DockablePanes, and subscribe to UI-level events. After OnStartup() completes, the ribbon structure is frozen — you cannot add new tabs or panels at runtime.
🟢 Key Methods
🟢public void CreateRibbonTab(string tabName)
Creates a new custom ribbon tab. The tab appears in the Revit ribbon between the standard Revit tabs and the Add-Ins tab. Tab names must be unique — if the tab already exists (e.g. from a previous Revit session), Revit throws. Always wrap in try/catch.
🟢public RibbonPanel CreateRibbonPanel(string tabName, string panelName)
Creates a panel on a specific tab. Use after CreateRibbonTab() for your custom tabs. Panel names must be unique within a tab.
🟢public RibbonPanel CreateRibbonPanel(string panelName)
Creates a panel on the built-in "Add-Ins" tab. Quick option when you don't need a custom tab.
🟢public void RegisterDockablePane(DockablePaneId id, string title, IDockablePaneProvider provider)
Registers a dockable pane. Must be called in OnStartup() before any pane operations. The provider creates the WPF content. See DockablePane section for the full lifecycle.
// Example — Complete tab + panel construction in OnStartup
public Result OnStartup(UIControlledApplication app)
{
    // ── Step 1: Create custom tab ──
    const string TAB   = "Studio Tools";
    const string DLL   = System.Reflection.Assembly
        .GetExecutingAssembly().Location;

    try { app.CreateRibbonTab(TAB); }
    catch { /* tab already exists — ignore */ }

    // ── Step 2: Create panels on the tab ──
    RibbonPanel panelModel = app.CreateRibbonPanel(TAB, "Model Tools");
    RibbonPanel panelView  = app.CreateRibbonPanel(TAB, "View Tools");
    RibbonPanel panelData  = app.CreateRibbonPanel(TAB, "Data Tools");

    // ── Step 3: Add buttons to panels ──
    BuildModelPanel(panelModel, DLL);
    BuildViewPanel(panelView, DLL);
    BuildDataPanel(panelData, DLL);

    return Result.Succeeded;
}
ClassPushButton
Inherits: RibbonButton → RibbonItem
Namespace: Autodesk.Revit.UI
A standard clickable button that triggers one IExternalCommand. The most common ribbon control. Created via panel.AddItem(PushButtonData). After creation you can update its text, image, tooltip, and availability class at any time.
🟠 Key Properties (runtime update)
🟠public string ItemText { get; set; }
Gets or sets the button label. Can be updated at runtime — e.g. changing "Show Panel" to "Hide Panel".
🟠public ImageSource LargeImage { get; set; }
32×32 button image (shown when panel is wide). Set with BitmapImage or BitmapFrame from WPF.
🟠public ImageSource Image { get; set; }
16×16 small button image (shown in stacked items or narrow panels).
🟠public string ToolTip { get; set; }
Short tooltip text shown on hover. Keep under 50 characters.
🟠public string LongDescription { get; set; }
Expanded tooltip shown after 1 second hover. Use to explain what the button does in detail.
🟠public bool Enabled { get; set; }
Enables or disables the button at runtime. Disabled buttons are greyed out. Prefer IExternalCommandAvailability for context-sensitive enabling.
🟠public bool Visible { get; set; }
Shows or hides the button. Hidden buttons don't occupy ribbon space.
ClassPushButtonData
Inherits: ButtonData → RibbonItemData
Namespace: Autodesk.Revit.UI
Data class used to configure a PushButton before adding it to a panel. Contains the command class name, assembly path, and all display properties. Create PushButtonData, configure it, then pass to panel.AddItem().
🟣 Constructor
🟣public PushButtonData(string name, string text, string assemblyName, string className)
Creates button data. name = internal unique ID (no spaces), text = display label, assemblyName = full DLL path, className = fully qualified class name of the IExternalCommand.
ParameterTypeDescription
namestringInternal unique name. No spaces. Used as the button's programmatic ID.
textstringDisplay label on the button. Use \n to force a line break.
assemblyNamestringFull absolute path to the .dll containing the command class.
classNamestringFully qualified class name: "Namespace.ClassName"
// Example — Complete PushButton with all properties
string dll = System.Reflection.Assembly.GetExecutingAssembly().Location;

// Helper: load image from embedded resource
System.Windows.Media.ImageSource LoadImg(string resourcePath)
{
    var uri = new Uri($"pack://application:,,,/{resourcePath}");
    return new System.Windows.Media.Imaging.BitmapImage(uri);
}

// ── Create PushButtonData ──
var btnData = new PushButtonData(
    "btn_WallCount",                    // internal name (no spaces)
    "Count\nWalls",                     // label (\n = line break)
    dll,                               // DLL path
    "StudioTools.WallCountCommand");    // fully qualified class

// Display properties
btnData.LargeImage   = LoadImg("StudioTools;component/Resources/wall_count_32.png");
btnData.Image        = LoadImg("StudioTools;component/Resources/wall_count_16.png");
btnData.ToolTip      = "Count all walls in the active document";
btnData.LongDescription = "Counts walls by type and outputs a summary report. "
                         + "Works on the active document or all open documents.";

// Availability — only enabled when a document is open
btnData.AvailabilityClassName = "StudioTools.RequiresDocument";

// Help file link (optional)
btnData.SetContextualHelp(new ContextualHelp(
    ContextualHelpType.Url,
    "https://docs.mystudio.com/wall-count"));

// ── Add to panel ──
PushButton btn = panelModel.AddItem(btnData) as PushButton;

// ── Runtime updates ──
btn.Enabled = false;                  // disable
btn.ItemText = "Wall\nReport";        // change label
btn.LargeImage = LoadImg("...");      // change icon
btn.ToolTip    = "Updated tooltip";   // change tooltip

// ── Panel separator (visual divider) ──
panelModel.AddSeparator();

// ── Slide-out panel (hidden controls below ──) ──
panelModel.AddSlideOut();              // items added after this go to the slide-out
panelModel.AddItem(anotherBtnData);    // this button only shows when expanded
ClassSplitButton
Inherits: RibbonItem
Namespace: Autodesk.Revit.UI
A button that shows the last-used command on the main face, with a drop-down arrow revealing other commands. The clicked command becomes the new "current" command shown on the face. Ideal for related commands where one is used most often (e.g. "Create Wall" variants).
🟢 Creation + Usage
// Example — SplitButton with 3 commands
// Create SplitButtonData
var splitData = new SplitButtonData("splitWallCreate", "Create Wall");

// Add to panel — returns SplitButton
SplitButton splitBtn = panel.AddItem(splitData) as SplitButton;

// Add commands to the split button's drop-down
PushButton pb1 = splitBtn.AddPushButton(new PushButtonData(
    "createBasicWall", "Basic Wall", dll, "StudioTools.CreateBasicWall"));
pb1.LargeImage = LoadImg("...");

PushButton pb2 = splitBtn.AddPushButton(new PushButtonData(
    "createCurtainWall", "Curtain Wall", dll, "StudioTools.CreateCurtainWall"));
pb2.LargeImage = LoadImg("...");

PushButton pb3 = splitBtn.AddPushButton(new PushButtonData(
    "createStackedWall", "Stacked Wall", dll, "StudioTools.CreateStackedWall"));
pb3.LargeImage = LoadImg("...");

// IsSynchronizedWithCurrentItem controls whether the face shows last-used command
splitBtn.IsSynchronizedWithCurrentItem = true;  // default and recommended

// Set default (current) button programmatically
splitBtn.CurrentButton = pb1;
ClassPulldownButton
Inherits: RibbonItem
Namespace: Autodesk.Revit.UI
A button with a permanent drop-down arrow that always shows a menu of commands. Unlike SplitButton, the face never changes — it always shows the same icon and text. Ideal for grouping a category of related but equally important commands (e.g. an "Export" button with PDF, DWG, IFC options).

ClassComboBox
Inherits: RibbonItem
Namespace: Autodesk.Revit.UI
A dropdown list control that shows text items. The user can select an item from the list or type a value. Items can be grouped using ComboBoxMember.GroupName. The CurrentChanged event fires when selection changes — useful for filter dropdowns, phase selectors, or view template pickers.
🟢 Creation + Events + Runtime Update
// Example — ComboBox phase selector updated on document open
// ── Create ComboBox in OnStartup ──
var cbData = new ComboBoxData("cbPhase");
cbData.ToolTip = "Select active phase";

static ComboBox _phaseComboBox;
_phaseComboBox = panel.AddItem(cbData) as ComboBox;

// Add static items at startup
_phaseComboBox.AddItem(new ComboBoxMemberData("existing", "Existing"));
_phaseComboBox.AddItem(new ComboBoxMemberData("newconstruction", "New Construction"));
_phaseComboBox.AddItem(new ComboBoxMemberData("demolition", "Demolition"));

// Subscribe to selection change event
_phaseComboBox.CurrentChanged += OnPhaseComboChanged;

// ── Update items dynamically when document opens ──
private void OnDocumentOpened(object s, DocumentOpenedEventArgs e)
{
    Document doc = e.Document;

    // Get all phases from document
    var phases = new FilteredElementCollector(doc)
        .OfClass(typeof(Phase)).Cast<Phase>()
        .OrderBy(p => p.Id.IntegerValue).ToList();

    // Clear and repopulate ComboBox
    _phaseComboBox.ClearList();
    foreach (var phase in phases)
    {
        var mbr = new ComboBoxMemberData(
            phase.Id.IntegerValue.ToString(),
            phase.Name);
        _phaseComboBox.AddItem(mbr);
    }
    // Set to last phase
    _phaseComboBox.Current = _phaseComboBox.GetItems().LastOrDefault();
}

// ── Handle selection change ──
private void OnPhaseComboChanged(object s, Autodesk.Revit.UI.Events.ComboBoxCurrentChangedEventArgs e)
{
    ComboBoxMember selected = e.NewValue;
    if (selected == null) return;

    System.Diagnostics.Debug.WriteLine($"Phase changed to: {selected.Name}");
    // Store selection for use by commands
    MyApp.SelectedPhaseId = new ElementId(int.Parse(selected.Name));
}
ClassTextBox
Inherits: RibbonItem
Namespace: Autodesk.Revit.UI
An editable text input field embedded in the ribbon. Used for quick filter inputs, search boxes, or numeric entry that commands can read. The EnterPressed and CurrentChanged events fire when the user presses Enter or changes the text.
🟢 Creation + Events
// Example — TextBox search filter in ribbon
var tbData = new TextBoxData("tbSearch");
tbData.Name        = "Search Rooms";
tbData.ToolTip     = "Filter rooms by name or number";
tbData.PromptText  = "Search rooms…";  // placeholder text
tbData.ShowImageAsButton = false;

static TextBox _searchBox;
_searchBox = panel.AddItem(tbData) as TextBox;

// Set width
_searchBox.Width = 140;   // pixels

// Enter pressed event
_searchBox.EnterPressed += (sender, args) =>
{
    string query = (_searchBox.Value as string) ?? "";
    System.Diagnostics.Debug.WriteLine($"Search: {query}");
    // Trigger a search command with the query
    MyApp.PendingSearchQuery = query;
    // Raise external event to execute the search
    MyApp.SearchEvent.Raise();
};

// Read value from a command
string currentSearch = _searchBox.Value as string;
ClassRadioButtonGroup
Inherits: RibbonItem
Namespace: Autodesk.Revit.UI
A group of mutually exclusive toggle buttons. Exactly one button in the group is active at any time. Used for mode selection (e.g. "Metric / Imperial", "Gross / Net areas", "2D / 3D view"). The CurrentChanged event fires when selection changes.
🟢 Creation + Usage
// Example — Unit mode toggle
var rbgData = new RadioButtonGroupData("rbgUnits");
RadioButtonGroup rbg = panel.AddItem(rbgData) as RadioButtonGroup;

// Add toggle buttons
ToggleButton metricBtn = rbg.AddItem(new ToggleButtonData(

ToggleButton imperialBtn = rbg.AddItem(new ToggleButtonData(

imperialBtn.LargeImage = LoadImg("imperial_32.png");

// Set default selection
rbg.Current = metricBtn;

// Handle change
rbg.CurrentChanged += (s, e) =>
{
    bool isMetric = rbg.Current.Name == "rbMetric";
    MyApp.UseMetric = isMetric;
    System.Diagnostics.Debug.WriteLine($"Unit mode: {(isMetric ? "Metric" : "Imperial")}");
};

IExternalCommandAvailability · PostableCommand

Control when ribbon buttons are enabled, and trigger built-in Revit commands programmatically.

InterfaceIExternalCommandAvailability
Namespace: Autodesk.Revit.UI
Controls whether a ribbon button is enabled or disabled based on the current Revit context. Implement this interface to make buttons context-sensitive — e.g. disabled when no document is open, or only active when a 3D view is active, or only when walls are selected. Set via PushButtonData.AvailabilityClassName.
// Performance NoteIsCommandAvailable() is called very frequently — on every selection change, view activation, and idle cycle. Keep it fast: no FilteredElementCollector queries inside. Only check UIApplication.ActiveUIDocument and UIDocument.ActiveGraphicalView.
🟢 Complete Availability Patterns
// Example — 5 common availability implementations
// ── 1. Requires any document open ──
public class RequiresDocument : IExternalCommandAvailability
{
    public bool IsCommandAvailable(UIApplication app,
        CategorySet selectedCategories)
    {
        return app.ActiveUIDocument != null;
    }
}

// ── 2. Requires a project document (not family) ──
public class RequiresProject : IExternalCommandAvailability
{
    public bool IsCommandAvailable(UIApplication app,
        CategorySet selectedCategories)
    {
        var doc = app.ActiveUIDocument?.Document;
        return doc != null && !doc.IsFamilyDocument;
    }
}

// ── 3. Only in a 3D view ──
public class Requires3DView : IExternalCommandAvailability
{
    public bool IsCommandAvailable(UIApplication app,
        CategorySet selectedCategories)
    {
        return app.ActiveUIDocument?.ActiveGraphicalView
            is View3D;
    }
}

// ── 4. Requires at least one wall selected ──
public class RequiresWallSelection : IExternalCommandAvailability
{
    public bool IsCommandAvailable(UIApplication app,
        CategorySet selectedCategories)
    {
        // selectedCategories — categories of currently selected elements
        // Fast — already computed by Revit, no collector needed
        foreach (Category cat in selectedCategories)
            if (cat.Id.IntegerValue == (int)BuiltInCategory.OST_Walls)
                return true;
        return false;
    }
}

// ── 5. Requires workshared document ──
public class RequiresWorkshared : IExternalCommandAvailability
{
    public bool IsCommandAvailable(UIApplication app,
        CategorySet selectedCategories)
    {
        var doc = app.ActiveUIDocument?.Document;
        return doc != null && doc.IsWorkshared;
    }
}

// ── Assign to button ──
btnData.AvailabilityClassName = typeof(RequiresWallSelection).FullName;
// Or as string: "StudioTools.RequiresWallSelection"

ClassRevitCommandId
Inherits: APIObject
Namespace: Autodesk.Revit.UI
Represents a reference to a built-in Revit command. Obtained from RevitCommandId.LookupPostableCommandId() or RevitCommandId.LookupCommandId(). Used with UIApplication.PostCommand() to trigger commands and with CanExecuteNextCommand to check availability.
🟢 Complete PostCommand Patterns
// Example — PostCommand usage patterns
UIApplication uiapp = commandData.Application;

// ── Method 1: From PostableCommand enum (most common) ──
RevitCommandId saveId = RevitCommandId
    .LookupPostableCommandId(PostableCommand.Save);

if (uiapp.CanPostCommand(saveId))
    uiapp.PostCommand(saveId);

// ── Method 2: From command name string ──
// Get command names by running the journal or using RevitCommandId.LookupCommandId
RevitCommandId purgeId = RevitCommandId
    .LookupCommandId("ID_PURGE_UNUSED");
if (purgeId != null && uiapp.CanPostCommand(purgeId))
    uiapp.PostCommand(purgeId);

// ── Practical examples ──

// Trigger Sync to Central after batch operation
var syncId = RevitCommandId.LookupPostableCommandId(
    PostableCommand.SynchronizeWithCentral);
if (uiapp.CanPostCommand(syncId))
    uiapp.PostCommand(syncId);

// Reset temporary hide/isolate after batch visual changes
var resetId = RevitCommandId.LookupPostableCommandId(
    PostableCommand.ResetTemporaryHideIsolate);
uiapp.PostCommand(resetId);

// Open a new 3D view
var v3dId = RevitCommandId.LookupPostableCommandId(
    PostableCommand.ThreeDView);
uiapp.PostCommand(v3dId);

// IMPORTANT: PostCommand is asynchronous!
// The command fires AFTER your current Execute() returns.
// You cannot wait for its result in the same Execute() call.
// If you need to react to it, subscribe to the relevant event.
PatternContextual Ribbon Tabs
Namespace: Autodesk.Revit.UI
Contextual tabs appear and disappear automatically based on user selection — exactly like Revit's own "Modify | Walls" tab. Implemented by subscribing to UIApplication.ViewActivated and Selection.SelectionChanged and showing/hiding panel visibility via RibbonPanel.Visible.
🟢 Contextual Tab Pattern
// Example — Show/hide panel based on selected element type
// In OnStartup — create panel but start hidden
RibbonPanel wallContextPanel = app.CreateRibbonPanel("Studio Tools", "Wall Tools");
wallContextPanel.Visible = false;   // hidden by default

// Store reference statically
MyApp.WallContextPanel = wallContextPanel;

// Subscribe to selection change via Idling (selection API)
app.Idling += OnIdlingCheckSelection;

// ── Handle selection changes ──
private static ICollection<ElementId> _lastSelection = new List<ElementId>();

private void OnIdlingCheckSelection(object sender, IdlingEventArgs e)
{
    e.SetRaiseWithoutDelay(false);   // be CPU-kind

    var uiapp = sender as UIApplication;
    var uidoc = uiapp?.ActiveUIDocument;
    if (uidoc == null) { MyApp.WallContextPanel.Visible = false; return; }

    var selIds = uidoc.Selection.GetElementIds();

    // Only update if selection changed
    if (selIds.SequenceEqual(_lastSelection)) return;
    _lastSelection = selIds;

    // Check if any selected element is a Wall
    bool hasWall = selIds
        .Select(id => uidoc.Document.GetElement(id))
        .Any(el => el is Wall);

    // Show/hide the contextual panel
    MyApp.WallContextPanel.Visible = hasWall;
}

DockablePane — Modeless Panel

A persistent WPF panel that docks inside the Revit window — the same mechanism Revit uses for Properties, Project Browser, and Systems Browser.

// DockablePane vs WPF WindowUse DockablePane for persistent tools that stay visible while the user works (e.g. a properties inspector, a room schedule panel, a clash report). Use regular WPF Window for modal/temporary dialogs that the user opens, uses, and closes.
1

Implement IDockablePaneProvider

Your WPF UserControl class implements this interface and provides its own content via SetupDockablePane().

2

Register in OnStartup()

Call app.RegisterDockablePane(paneId, title, provider) once. Cannot be done after startup.

3

Show/Hide at runtime

Call uiapp.GetDockablePane(paneId).Show() / .Hide() from any command or event handler.

DirectContext3D

Draw custom 3D geometry directly into Revit 3D views in real-time — without creating elements. Used for clash highlights, live analysis overlays, progress indicators, and custom geometry previews.

// What is DirectContext3D?DirectContext3D lets you bypass the Revit element system entirely and draw arbitrary OpenGL-style geometry (triangles, lines, points) directly into any 3D view at render time. The geometry is NOT stored in the model — it's drawn fresh on every frame. Perfect for visualisations that change frequently or don't belong in the model.
InterfaceIDirectContext3DServer
Inherits: IExternalServer
Namespace: Autodesk.Revit.DB.DirectContext3D
The core interface for custom 3D rendering in Revit views. Implement this interface, register it with ExternalServiceRegistry, and Revit will call RenderScene() on every view redraw. Your implementation creates vertex/index buffers and submits draw calls to the DrawContext.
🟢 Complete IDirectContext3DServer Implementation
// Example — Clash highlight server: draw red transparent boxes on clashes
using Autodesk.Revit.DB.DirectContext3D;
using Autodesk.Revit.DB.ExternalService;

public class ClashHighlightServer : IDirectContext3DServer
{
    private Guid _serviceId = ExternalServices.BuiltInExternalServices.DirectContext3DService;
    private Guid _serverId  = new Guid("CCDD1122-3344-5566-7788-AABBCCDD1122");

    // External list of bounding boxes to highlight as clash volumes
    public List<BoundingBoxXYZ> ClashBoxes { get; set; } = new();

    // ── IExternalServer required methods ──
    public Guid   GetServiceId()      => _serviceId;
    public Guid   GetServerId()       => _serverId;
    public string GetVendorId()        => "STUDIO";
    public string GetName()            => "Clash Highlight Server";
    public string GetDescription()     => "Draws clash volumes as red boxes";
    public string GetApplicationId()   => "StudioTools";

    // ── IDirectContext3DServer core methods ──

    // Should this server draw in the given view?
    public bool CanExecute(View dBView)
    {
        // Only draw in 3D views
        return dBView is View3D && ClashBoxes.Count > 0;
    }

    // Return outline that encompasses all your custom geometry
    // Used for view clipping and level-of-detail decisions
    public Outline GetBoundingBox(View dBView)
    {
        if (ClashBoxes.Count == 0)
            return new Outline(XYZ.Zero, XYZ.Zero);

        XYZ min = new XYZ(
            ClashBoxes.Min(b => b.Min.X),
            ClashBoxes.Min(b => b.Min.Y),
            ClashBoxes.Min(b => b.Min.Z));
        XYZ max = new XYZ(
            ClashBoxes.Max(b => b.Max.X),
            ClashBoxes.Max(b => b.Max.Y),
            ClashBoxes.Max(b => b.Max.Z));
        return new Outline(min, max);
    }

    // Does your geometry need to be re-rendered every frame?
    public bool UseInTransparentPass(View dBView) => true; // transparent = semi-opaque

    // ── The main draw method — called every frame ──
    public void RenderScene(View dBView, DisplayStyle displayStyle)
    {
        try
        {
            foreach (var bb in ClashBoxes)
                DrawBox(bb, new ColorWithTransparency(255, 0, 0, 128));  // red, 50% transparent
        }
        catch (Exception ex)
        {
            System.Diagnostics.Debug.WriteLine($"DC3D render error: {ex.Message}");
        }
    }

    // Helper: draw a wireframe + transparent solid box
    private static void DrawBox(BoundingBoxXYZ bb, ColorWithTransparency color)
    {
        XYZ mn = bb.Min, mx = bb.Max;

        // 8 vertices of the box
        XYZ[] v =
        {
            new XYZ(mn.X,mn.Y,mn.Z), new XYZ(mx.X,mn.Y,mn.Z),
            new XYZ(mx.X,mx.Y,mn.Z), new XYZ(mn.X,mx.Y,mn.Z),
            new XYZ(mn.X,mn.Y,mx.Z), new XYZ(mx.X,mn.Y,mx.Z),
            new XYZ(mx.X,mx.Y,mx.Z), new XYZ(mn.X,mx.Y,mx.Z)
        };

        // Build vertex buffer (position + colour per vertex)
        VertexFormatBits vfBits = VertexFormatBits.PositionColored;
        int vertexCount = v.Length;  // 8
        VertexBufferStorage vbs = new VertexBufferStorage(vfBits, vertexCount);

        VertexBuffer vb = vbs.VertexBuffer;
        vb.Map(vertexCount * VertexPositionColored.GetSizeInFloats());
        {
            VertexStreamPositionColored vs = vb.GetVertexStreamPositionColored();
            foreach (var pt in v)
                vs.AddVertex(new VertexPositionColored(pt, color));
        }
        vb.Unmap();

        // 12 triangles for 6 faces (2 triangles per face)
        int[] faces = {
            0,1,2, 0,2,3,  // bottom
            4,5,6, 4,6,7,  // top
            0,1,5, 0,5,4,  // front
            2,3,7, 2,7,6,  // back
            0,3,7, 0,7,4,  // left
            1,2,6, 1,6,5   // right
        };
        IndexBuffer ib = new IndexBuffer(faces.Length);
        ib.Map(faces.Length);
        {
            IndexStream ixs = ib.GetIndexStreamTriangle();
            for (int i = 0; i < faces.Length; i += 3)
                ixs.AddTriangle(new IndexTriangle(faces[i], faces[i+1], faces[i+2]));
        }
        ib.Unmap();

        // Submit draw call to Revit's render pipeline
        var effect = new EffectInstance(vfBits);
        DrawContext.FlushBuffer(vb, vertexCount, ib,
            faces.Length / 3,
            new VertexFormat(vfBits),
            effect,
            PrimitiveType.TriangleList);
    }
}
