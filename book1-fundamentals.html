Application & Document

Every command starts with ExternalCommandData. From it you navigate down to the document and model elements.

Object Hierarchy

hierarchy
UIApplication          // Revit application + UI layer
  └── UIDocument       // Active open document (with selection)
        └── Document   // The Revit project/family file (.rvt/.rfa)
              └── Element  // Any object in the model

Getting the Document — Standard Pattern

C#
public Result Execute(ExternalCommandData commandData,
    ref string message, ElementSet elements)
{
    // Application layer
    UIApplication uiapp = commandData.Application;
    Application   app   = uiapp.Application;

    // Document layer
    UIDocument uidoc = uiapp.ActiveUIDocument;
    Document   doc   = uidoc.Document;

    // Document info
    string title    = doc.Title;
    bool   isFam    = doc.IsFamilyDocument;
    string filePath = doc.PathName;

    return Result.Succeeded;
}

Key Document Members

MemberTypeDescription
doc.TitlestringFile name without extension
doc.PathNamestringFull file path. Empty if unsaved
doc.IsFamilyDocumentboolTrue if this is a .rfa family file
doc.ActiveViewViewCurrently active view
doc.ApplicationApplicationThe Revit Application object
doc.CreateCreation.DocumentFactory for creating elements
doc.FamilyCreateCreation.FamilyItemFactoryFactory for family geometry (family docs only)
doc.GetElement(id)ElementLook up element by ElementId
doc.Delete(id)ICollection<ElementId>Delete element, returns all deleted ids
doc.Save()voidSave the document
doc.IsModifiedboolWhether document has unsaved changes
// Opening Other Documents Use uiapp.Application.OpenDocumentFile(filePath) to open another .rvt. Use doc.LoadFamily(path, out family) to load a .rfa into the current project.
// 02

Element Hierarchy

Everything in Revit is an Element. Understanding the type hierarchy is critical for filtering and casting correctly.

hierarchy
Element
  ├── Instance elements    // physical objects placed in model
  │     ├── Wall, Floor, Ceiling, Roof
  │     ├── FamilyInstance  // doors, windows, furniture, columns
  │     ├── Room, Space, Area
  │     └── MEP: Pipe, Duct, CableTray, Conduit
  ├── Type elements        // define the 'blueprint'
  │     ├── WallType, FloorType, RoofType
  │     ├── FamilySymbol    // a specific size/variant of a family
  │     └── Family          // the family container
  └── Non-geometric        // configuration / meta
        ├── Level, Grid, ReferencePlane
        ├── View, ViewSheet, ViewPlan
        └── Material, LineStyle

Accessing Element Properties

C#
Element elem = doc.GetElement(new ElementId(12345));

// Universal properties on every element
ElementId id       = elem.Id;
string    name     = elem.Name;
Category  cat      = elem.Category;
ElementId typeId   = elem.GetTypeId();
Level     level    = doc.GetElement(elem.LevelId) as Level;

// Get the element type
ElementType type = doc.GetElement(elem.GetTypeId()) as ElementType;

// Check the category
if (elem.Category.Id.IntegerValue == (int)BuiltInCategory.OST_Walls)
{
    Wall wall = elem as Wall;
}

// Bounding box (null if element has no geometry)
BoundingBoxXYZ bb = elem.get_BoundingBox(doc.ActiveView);
// Instance vs Type Instance elements are placed in the model. Type elements define their template. A FamilyInstance (door) references a FamilySymbol (door type) which belongs to a Family (door family). Always check both instance AND type parameters.
// 03

Parameters

Parameters are the data attached to elements. There are four types and five ways to access them.

🔧
Built-in
Hardcoded by Autodesk. Most reliable. Use BuiltInParameter enum.
BuiltInParameter
📋
Project
Created by user in project. Access by name or GUID.
LookupParameter()
🔗
Shared
Defined in external .txt file. Access by GUID (most stable).
get_Parameter(Guid)
🌐
Global
Document-level parameters not attached to a category.
GlobalParametersManager

Reading Parameters

C#
Element elem = ...;

// Method 1: BuiltIn (BEST — language independent, fast)
Parameter p1 = elem.get_Parameter(BuiltInParameter.ALL_MODEL_MARK);

// Method 2: By Name (fragile — breaks with language change)
Parameter p2 = elem.LookupParameter("Mark");

// Method 3: By GUID (shared params — very stable)
Parameter p3 = elem.get_Parameter(new Guid("your-guid-here"));

// Method 4: Get all parameters
ParameterSet allParams = elem.Parameters;

// Reading values — check StorageType first!
if (p1 != null)
{
    switch (p1.StorageType)
    {
        case StorageType.String:
            string val = p1.AsString();          break;
        case StorageType.Double:
            double dbl = p1.AsDouble();          break;
        case StorageType.Integer:
            int    num = p1.AsInteger();         break;
        case StorageType.ElementId:
            ElementId eid = p1.AsElementId();   break;
    }

    // AsValueString() always returns a human-readable string
    string display = p1.AsValueString();
}

Setting Parameters

C#
// Always inside a Transaction!
using (var tx = new Transaction(doc, "Set Parameter"))
{
    tx.Start();

    Parameter p = wall.get_Parameter(BuiltInParameter.ALL_MODEL_MARK);

    if (p != null && !p.IsReadOnly)
    {
        p.Set("W-001");         // string
        // p.Set(3.5);           // double (internal units = feet)
        // p.Set(1);             // integer (or enum int value)
        // p.Set(new ElementId(x)); // ElementId
    }

    tx.Commit();
}

Parameter StorageType Reference

StorageTypeRead MethodSet MethodNotes
StringAsString()Set(string)Text values. AsValueString() also works.
DoubleAsDouble()Set(double)Always in internal units (feet). Convert with UnitUtils.
IntegerAsInteger()Set(int)Also used for Yes/No (1/0) and enums.
ElementIdAsElementId()Set(ElementId)References to other elements (material, level, etc).
// Unit Conversion All double parameters are stored in internal units (feet). To set 3000mm: p.Set(UnitUtils.ConvertToInternalUnits(3000, UnitTypeId.Millimeters)). To read in mm: UnitUtils.ConvertFromInternalUnits(p.AsDouble(), UnitTypeId.Millimeters).
// 04

Filtering Elements

FilteredElementCollector is the main query engine. Always filter with native (quick) filters first, LINQ second.

Quick Filters (Fast — run in Revit's native engine)

C#
// By Class (fastest — use when possible)
var walls = new FilteredElementCollector(doc)
    .OfClass(typeof(Wall))
    .Cast<Wall>()
    .ToList();

// By Category
var doors = new FilteredElementCollector(doc)
    .OfCategory(BuiltInCategory.OST_Doors)
    .WhereElementIsNotElementType()   // instances only
    .ToList();

// Element Types only (families, wall types etc.)
var wallTypes = new FilteredElementCollector(doc)
    .OfClass(typeof(WallType))
    .Cast<WallType>()
    .ToList();

// In a specific view (only elements visible in that view)
var inView = new FilteredElementCollector(doc, doc.ActiveView.Id)
    .OfCategory(BuiltInCategory.OST_Walls)
    .ToList();

// At a specific level
var levelFilter = new ElementLevelFilter(level.Id);
var onLevel = new FilteredElementCollector(doc)
    .WherePasses(levelFilter)
    .OfCategory(BuiltInCategory.OST_Walls)
    .ToList();

// Multiple categories at once
var cats = new List<BuiltInCategory>
    { BuiltInCategory.OST_Walls, BuiltInCategory.OST_Floors };
var multiCat = new FilteredElementCollector(doc)
    .WherePasses(new ElementMulticategoryFilter(cats))
    .ToList();

Parameter Filters (Slow — but powerful)

C#
// Filter walls where Mark contains "EXT"
var provider  = new ParameterValueProvider(
    new ElementId(BuiltInParameter.ALL_MODEL_MARK));
var evaluator = new FilterStringContains();
var rule      = new FilterStringRule(provider, evaluator, "EXT");

var paramFilter = new ElementParameterFilter(rule);

var extWalls = new FilteredElementCollector(doc)
    .OfClass(typeof(Wall))
    .WherePasses(paramFilter)
    .Cast<Wall>()
    .ToList();

Retrieval Methods

MethodReturnsUse When
.ToElements()IList<Element>You need the elements themselves (most common)
.ToElementIds()ICollection<ElementId>You only need IDs (faster, less memory)
.FirstElement()ElementYou only need one result
.FirstOrDefault()ElementLINQ — first or null
.Cast<T>()IEnumerable<T>After OfClass() — gives typed results
.Count()intJust count — don't load elements
// Performance Rule Chain quick filters first: .OfClass() → .OfCategory() → .WhereElementIsNotElementType() then apply LINQ .Where() last. Never call .ToList() mid-chain before all native filters are applied.

Selection

Get elements the user has selected, or prompt them to pick elements interactively.

C#
Selection sel = uidoc.Selection;

// 1. Get currently selected elements
ICollection<ElementId> selectedIds = sel.GetElementIds();
foreach (var id in selectedIds)
{
    Element e = doc.GetElement(id);
}

// 2. Pick a single element
Reference pickedRef = sel.PickObject(
    ObjectType.Element, "Select an element");
Element picked = doc.GetElement(pickedRef);

// 3. Pick a specific type (e.g. Wall only)
Reference wallRef = sel.PickObject(
    ObjectType.Element,
    new WallSelectionFilter(),   // custom ISelectionFilter
    "Select a wall");

// 4. Pick multiple elements
IList<Reference> multiRefs = sel.PickObjects(
    ObjectType.Element, "Select walls");

// 5. Pick a point in the model
XYZ pt = sel.PickPoint("Click a point");

// 6. Set selection programmatically
var idsToSelect = walls.Select(w => w.Id).ToList();
sel.SetElementIds(idsToSelect);

Custom Selection Filter

C#
// Implement ISelectionFilter to restrict what the user can pick
public class WallSelectionFilter : ISelectionFilter
{
    public bool AllowElement(Element elem)
        => elem is Wall;    // only walls can be picked

    public bool AllowReference(Reference reference, XYZ position)
        => true;
}
// 06

Transactions

Any modification to the Revit model MUST happen inside a Transaction. This is non-negotiable.

Transaction Types

📝
Transaction
Single operation. Start → do work → Commit or RollBack.
most common
📦
TransactionGroup
Wraps multiple Transactions into one undo step for the user.
batch operations
🔄
SubTransaction
Nested within an open Transaction. Can be rolled back independently.
partial rollback

Transaction Patterns

C#
// Pattern 1: Basic (using = auto-dispose)
using (var tx = new Transaction(doc, "My Operation"))
{
    tx.Start();
    // ... modify elements ...
    tx.Commit();
}

// Pattern 2: With error handling
using (var tx = new Transaction(doc, "Safe Operation"))
{
    tx.Start();
    try
    {
        // ... work ...
        tx.Commit();
    }
    catch (Exception ex)
    {
        tx.RollBack();
        TaskDialog.Show("Error", ex.Message);
    }
}

// Pattern 3: TransactionGroup (one undo step)
using (var tg = new TransactionGroup(doc, "Batch Update"))
{
    tg.Start();

    using (var tx1 = new Transaction(doc, "Step 1"))
    { tx1.Start(); /* ... */ tx1.Commit(); }

    using (var tx2 = new Transaction(doc, "Step 2"))
    { tx2.Start(); /* ... */ tx2.Commit(); }

    tg.Assimilate();  // merges all into one undo step
}

// Pattern 4: SubTransaction (partial rollback)
using (var tx = new Transaction(doc, "Main"))
{
    tx.Start();
    using (var sub = new SubTransaction(doc))
    {
        sub.Start();
        try { /* risky work */ sub.Commit(); }
        catch { sub.RollBack(); /* main tx continues */ }
    }
    tx.Commit();
}

Transaction Status

StatusMeaning
TransactionStatus.StartedTransaction is open and ready
TransactionStatus.CommittedChanges saved to model
TransactionStatus.RolledBackChanges discarded
TransactionStatus.ErrorSomething went wrong — check exceptions
// Golden Rules 1) Never start a Transaction inside an event handler that fires during a transaction. 2) Never use a Transaction from a background thread — use IExternalEventHandler instead. 3) Always check TransactionStatus after Commit() in critical flows.

Units

Revit stores ALL distances in decimal feet internally. You must convert when reading/writing length parameters.

C#
using Autodesk.Revit.DB;

// Convert FROM internal (feet) TO display unit
double internalValue = wall.get_Parameter(
    BuiltInParameter.WALL_USER_HEIGHT_PARAM).AsDouble();

double mm      = UnitUtils.ConvertFromInternalUnits(
    internalValue, UnitTypeId.Millimeters);
double meters  = UnitUtils.ConvertFromInternalUnits(
    internalValue, UnitTypeId.Meters);

// Convert TO internal (feet) FROM display unit
double feet = UnitUtils.ConvertToInternalUnits(3000, UnitTypeId.Millimeters);

// Simple manual conversion (legacy approach)
double mmVal    = internalValue * 304.8;
double feetVal  = mmVal / 304.8;

UnitTypeId.MetersLength in metres
