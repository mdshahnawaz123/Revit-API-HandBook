ClassFace
Inherits: GeometryObject
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents a face (surface) on a Revit solid. Faces can be planar (PlanarFace), cylindrical (CylindricalFace), conical, ruled, or Nurbs. Each Face provides access to its normal, area, UV parameters, edge loops, and a Reference for dimensioning. Always obtain faces through a Solid — Face cannot be instantiated directly.
// NoteFace.Reference is only valid when Options.ComputeReferences = true was set during get_Geometry(). Without this, Reference will be null and cannot be used for dimensioning.
🟠 Key Properties
🟠public Reference Reference { get; }
Gets a Reference to this face that can be used for dimensioning with doc.Create.NewDimension(). Only valid when Options.ComputeReferences = true was set.
🟠public double Area { get; }
Gets the surface area of this face in internal units (square feet). Convert to m² by multiplying by 0.092903.
🟢 Key Methods
🟢public XYZ ComputeNormal(UV uv)
Computes the outward normal vector of this face at the given UV parameter position. For a PlanarFace the normal is constant across the entire face. For curved faces the normal varies.
ParameterTypeDescription
uvUVThe UV parameter at which to compute the normal. Use (0.5, 0.5) for the face centre.
Returns: XYZ — Unit normal vector pointing outward from the solid.
// Example — Face normal and area extraction
Options opts = new Options { ComputeReferences = true };
GeometryElement geom = wall.get_Geometry(opts);

foreach (GeometryObject obj in geom)
{
    if (obj is Solid solid && solid.Volume > 0)
    {
        foreach (Face face in solid.Faces)
        {
            XYZ    normal  = face.ComputeNormal(new UV(.5, .5));
            double areaSqM = face.Area * 0.092903;
            Reference fRef = face.Reference;  // for dimensions

            // Classify face direction
            if (Math.Abs(normal.Z) > 0.9)
                System.Diagnostics.Debug.WriteLine($"Horizontal face: {areaSqM:F2}m²");
            else
                System.Diagnostics.Debug.WriteLine($"Vertical face: {areaSqM:F2}m²");
        }
    }
}
🟢public Mesh Triangulate()
Returns a triangulated mesh approximation of this face. Used for custom geometry export, rendering, area calculations, and 3D printing workflows.
Returns: Mesh — Triangulated mesh with Vertices and Triangles collections.
// Example — Triangulate a face for OBJ export
Face face = ...;
Mesh mesh = face.Triangulate();

IList<XYZ>   verts     = mesh.Vertices;
int          numTris  = mesh.NumTriangles;

for (int i = 0; i < numTris; i++)
{
    MeshTriangle tri = mesh.get_Triangle(i);
    XYZ v0 = tri.get_Vertex(0);
    XYZ v1 = tri.get_Vertex(1);
    XYZ v2 = tri.get_Vertex(2);
    // Write "f i0 i1 i2" to OBJ file
    System.Diagnostics.Debug.WriteLine(
        $"f {tri.get_Index(0)} {tri.get_Index(1)} {tri.get_Index(2)}");
}
🟢public IList<CurveLoop> GetEdgesAsCurveLoops()
Returns the boundary of this face as a list of CurveLoops — one outer loop and any inner loops (holes). Used for extracting face profiles for re-use as floor/ceiling boundaries or for computing perimeter.
Returns: IList<CurveLoop> — Index 0 = outer boundary; subsequent loops = holes.
🟢public IntersectionResultArray Intersect(Curve curve)
Finds intersections between this face and a curve. Returns each point where the curve crosses the face surface. Used for ray casting, pipe-to-floor intersection detection etc.
ClassFamilyInstance
Inherits: Instance → Element
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents a placed instance of a loaded family in the model — doors, windows, columns, furniture, equipment, light fixtures, mechanical equipment, plumbing fixtures etc. Every FamilyInstance has a FamilySymbol (type) and a Family. FamilyInstance is the most common class in large Revit models.
🟠 Key Properties
🟠public FamilySymbol Symbol { get; set; }
Gets or sets the FamilySymbol (type) of this instance. Setting changes the type and requires an open Transaction. The new symbol must be from the same Family.
🟠public Element Host { get; }
Gets the host element if this is a hosted family instance (door in wall, window in wall, light in ceiling). Returns null for unhosted (level-based) instances.
🟠public bool HandFlipped { get; }
Gets whether this instance has been hand-flipped (mirrored across the hand orientation plane). Relevant for asymmetric families like doors.
🟠public bool FacingFlipped { get; }
Gets whether this instance has been facing-flipped (mirrored across the facing orientation plane).
🟠public StructuralType StructuralType { get; }
Gets the structural role of this instance (NonStructural, Beam, Column, Brace, Footing). Used to distinguish structural from non-structural family instances.
🟢 Key Methods
🟢public IList<Reference> GetReferences(FamilyInstanceReferenceType referenceType)
Returns references to named reference planes within the family geometry. Used to get dimensionable references to specific planes like Left/Right/Front/Back/CenterLeftRight/CenterFrontBack.
ParameterTypeDescription
referenceTypeFamilyInstanceReferenceTypeLeft, Right, Front, Back, Top, Bottom, CenterLeftRight, CenterFrontBack, CenterElevation, StrongReference, WeakReference
// Example — FamilyInstance references for dimensioning
FamilyInstance door = ...;

// Get references to the left and right planes of the door
IList<Reference> leftRefs  = door.GetReferences(FamilyInstanceReferenceType.Left);
IList<Reference> rightRefs = door.GetReferences(FamilyInstanceReferenceType.Right);

if (leftRefs.Count > 0 && rightRefs.Count > 0)
{
    ReferenceArray refs = new ReferenceArray();
    refs.Append(leftRefs[0]);
    refs.Append(rightRefs[0]);

    LocationPoint lp = door.Location as LocationPoint;
    Line dimLine = Line.CreateBound(
        lp.Point + new XYZ(0, 2, 0),
        lp.Point + new XYZ(3, 2, 0));

    using (var tx = new Transaction(doc, "Dimension Door"))
    {
        tx.Start();
        doc.Create.NewDimension(doc.ActiveView, dimLine, refs);
        tx.Commit();
    }
}

// Navigate the family hierarchy
FamilySymbol sym    = door.Symbol;
Family       family = sym.Family;
string info = $"Family: {family.Name} | Type: {sym.Name}";

// Change the type
using (var tx = new Transaction(doc, "Change Type"))
{
    tx.Start();
    if (!newSym.IsActive) newSym.Activate();
    door.Symbol = newSym;
    tx.Commit();
}
ClassFamilySymbol
Inherits: ElementType → Element
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents a specific type within a loaded family. For example, a "900×2100 Single Flush" door type is a FamilySymbol within the "Single-Flush" Family. You must call Activate() before placing a FamilySymbol that has never been used in the model.
🟠 Key Properties
🟠public Family Family { get; }
Gets the Family that this symbol belongs to.
🟠public bool IsActive { get; }
Gets whether this symbol is active (has been loaded into the current session). Must be true before placing. Call Activate() if false.
🟢 Key Methods
🟢public void Activate()
Activates this FamilySymbol so it can be placed in the model. Must be called inside a Transaction before the first NewFamilyInstance() call if IsActive is false. Harmless to call even if already active.
// Example — Activate and place a family symbol
FamilySymbol sym = new FilteredElementCollector(doc)
    .OfClass(typeof(FamilySymbol))
    .OfCategory(BuiltInCategory.OST_Furniture)
    .Cast<FamilySymbol>()
    .FirstOrDefault(s => s.Name.Contains("Desk"));

if (sym == null) return Result.Failed;

Level level = new FilteredElementCollector(doc)
    .OfClass(typeof(Level)).Cast<Level>().First();

using (var tx = new Transaction(doc, "Place Desk"))
{
    tx.Start();
    if (!sym.IsActive) sym.Activate();   // Always check!

    FamilyInstance desk = doc.Create.NewFamilyInstance(
        new XYZ(5, 5, 0), sym, level,
        StructuralType.NonStructural);
    tx.Commit();
}
ClassFloor
Inherits: HostObject → Element
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents a floor slab element. Floors are created using Floor.Create() with a list of CurveLoops defining the boundary (outer loop + optional inner holes). Access the compound structure layers via FloorType.
🟢 Static Factory Methods
🟣public static Floor Create(Document doc, IList<CurveLoop> profile, ElementId floorTypeId, ElementId levelId)
Creates a floor from one or more CurveLoops. The first CurveLoop is the outer boundary; additional loops define holes/openings in the floor. All loops must be planar and co-planar.
ParameterTypeDescription
docDocumentThe document to create the floor in.
profileIList<CurveLoop>Boundary loops. [0] = outer profile, [1..n] = inner holes.
floorTypeIdElementIdThe ElementId of the FloorType to use.
levelIdElementIdThe ElementId of the Level this floor sits on.
Returns: Floor — The newly created Floor element.
// Example — Floor with a rectangular hole
FloorType ft = new FilteredElementCollector(doc)
    .OfClass(typeof(FloorType)).Cast<FloorType>().First();
Level lv = new FilteredElementCollector(doc)
    .OfClass(typeof(Level)).Cast<Level>().First();

// Outer boundary 10×8 metres
CurveLoop outer = new CurveLoop();
outer.Append(Line.CreateBound(new XYZ(0,0,0),  new XYZ(32.8,0,0)));
outer.Append(Line.CreateBound(new XYZ(32.8,0,0),new XYZ(32.8,26.2,0)));
outer.Append(Line.CreateBound(new XYZ(32.8,26.2,0),new XYZ(0,26.2,0)));
outer.Append(Line.CreateBound(new XYZ(0,26.2,0),new XYZ(0,0,0)));

// Inner hole 2×2 metres (stairwell opening)
CurveLoop hole = new CurveLoop();
hole.Append(Line.CreateBound(new XYZ(10,10,0),new XYZ(16.6,10,0)));
hole.Append(Line.CreateBound(new XYZ(16.6,10,0),new XYZ(16.6,16.6,0)));
hole.Append(Line.CreateBound(new XYZ(16.6,16.6,0),new XYZ(10,16.6,0)));
hole.Append(Line.CreateBound(new XYZ(10,16.6,0),new XYZ(10,10,0)));

using (var tx = new Transaction(doc, "Create Floor"))
{
    tx.Start();
    Floor floor = Floor.Create(doc,
        new List<CurveLoop> { outer, hole },
        ft.Id, lv.Id);
    tx.Commit();
}
ClassGrid
Inherits: DatumPlane → Element
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents a grid datum element. Grids are used as reference lines for structural and architectural coordination. They appear in plan views and can be dimensioned using new Reference(grid) — NOT grid.Curve.Reference.
// Critical — Grid ReferencesUse new Reference(grid) to get a reference to a grid for dimensioning. Do NOT use grid.Curve.Reference — that returns null for grids.
🟢 Static Factory + Methods
🟣public static Grid Create(Document doc, Line line)
Creates a straight grid from a line. The grid is infinite in plan view but clipped to the line endpoints as the default extent.
// Example — Grid system creation and dimensioning
using (var tx = new Transaction(doc, "Create Grids"))
{
    tx.Start();
    double spacing = UnitUtils.ConvertToInternalUnits(6000, UnitTypeId.Millimeters);

    // Create 4 vertical grids A-D
    string[] names = { "A", "B", "C", "D" };
    List<Grid> grids = new List<Grid>();
    for (int i = 0; i < 4; i++)
    {
        double x = i * spacing;
        Line line = Line.CreateBound(
            new XYZ(x, -5, 0), new XYZ(x, 25, 0));
        Grid g = Grid.Create(doc, line);
        g.Name = names[i];
        grids.Add(g);
    }

    // Dimension between grids using new Reference(grid)
    ReferenceArray refs = new ReferenceArray();
    foreach (var g in grids)
        refs.Append(new Reference(g));   // ← CORRECT

    Line dimLine = Line.CreateBound(
        new XYZ(0, -8, 0), new XYZ(3 * spacing, -8, 0));
    doc.Create.NewDimension(doc.ActiveView, dimLine, refs);

    tx.Commit();
}
🟢public Curve Curve { get; }
Gets the curve defining the grid line's position and extent. For straight grids this is a Line; for arc grids this is an Arc.
ClassJoinGeometryUtils
Inherits: System.Object (static utility)
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Static utility class for joining and unjoining element geometry. When elements are joined, Revit cuts the overlapping volume to show a proper geometric union (e.g. wall-to-slab, column-to-beam). Joining and unjoining requires a Transaction.
🟢 Static Methods
🟢public static void JoinGeometry(Document doc, Element firstElement, Element secondElement)
Joins the geometry of two intersecting elements. After joining, one element's solid is cut by the other based on join order. Both elements must intersect geometrically.
// Example — Join wall to floor and check status
Wall  wall  = ...;
Floor floor = ...;

using (var tx = new Transaction(doc, "Join"))
{
    tx.Start();

    if (!JoinGeometryUtils.AreElementsJoined(doc, wall, floor))
        JoinGeometryUtils.JoinGeometry(doc, wall, floor);

    // Swap join order so wall wins (cuts floor)
    if (JoinGeometryUtils.GetJoinedElements(doc, wall).Contains(floor.Id))
        JoinGeometryUtils.SwitchJoinOrder(doc, wall, floor);

    tx.Commit();
}
🟢public static bool AreElementsJoined(Document doc, Element firstElement, Element secondElement)
Returns true if the two elements are currently joined. Use before calling JoinGeometry() to avoid errors on already-joined pairs.
Returns: bool — true if joined.
🟢public static void SwitchJoinOrder(Document doc, Element firstElement, Element secondElement)
Reverses which element cuts the other. In a wall-to-column join, if the column is being cut by the wall, switching makes the wall be cut by the column instead.
🟢public static void UnjoinGeometry(Document doc, Element firstElement, Element secondElement)
Removes the join between two elements, restoring their independent geometry.
ClassLevel
Inherits: DatumPlane → Element
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents a horizontal datum plane (floor level) in the Revit model. All model elements are attached to a level. Levels drive floor plan views and define the reference elevation for all associated elements.
🟢 Static Factory + Properties
🟣public static Level Create(Document doc, double elevation)
Creates a new level at the specified elevation in internal units (feet). The level name defaults to "Level N" but should be set immediately after creation.
// Example — Create levels from floor heights
double[] floorHeightsMM = { 0, 3500, 7000, 10500, 14000 };
string[] levelNames     = { "Ground", "Level 1", "Level 2", "Level 3", "Roof" };

using (var tx = new Transaction(doc, "Create Levels"))
{
    tx.Start();
    for (int i = 0; i < floorHeightsMM.Length; i++)
    {
        double elevFt = UnitUtils.ConvertToInternalUnits(
            floorHeightsMM[i], UnitTypeId.Millimeters);
        Level lvl = Level.Create(doc, elevFt);
        lvl.Name = levelNames[i];
    }
    tx.Commit();
}
🟠public double Elevation { get; set; }
Gets or sets the level elevation in internal units (feet). Setting this moves all attached elements. Always convert from/to mm using UnitUtils.
ClassLine
Inherits: Curve → APIObject
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents a straight line in 3D space. The most common curve type — used for walls, CurveLoops, dimensions, grids, model lines, and detail lines. Created via static factory methods.
🟢 Static Factory Methods
🟣public static Line CreateBound(XYZ endpoint0, XYZ endpoint1)
Creates a bounded (finite) line between two points. This is the most commonly used Line factory method. Both endpoints are required; the line has a definite start and end.
// Example — Line usage patterns
// Bounded line (has start/end points)
Line bounded = Line.CreateBound(
    new XYZ(0, 0, 0),
    new XYZ(10, 0, 0));

// Unbound line (infinite — direction only)
Line infinite = Line.CreateUnbound(new XYZ(0,0,0), XYZ.BasisX);

// Line properties
XYZ    start  = bounded.GetEndPoint(0);
XYZ    end    = bounded.GetEndPoint(1);
double length = bounded.Length;
XYZ    dir    = bounded.Direction;   // unit vector start→end
XYZ    mid    = bounded.Evaluate(0.5, true);

// Project a point onto the line
XYZ pt = new XYZ(5, 3, 0);
IntersectionResult proj = bounded.Project(pt);
XYZ projected = proj.XYZPoint;
double dist = proj.Distance;
🟣public static Line CreateUnbound(XYZ origin, XYZ direction)
Creates an unbounded (infinite) line through origin in the given direction. Used for infinite geometry operations like plane intersections. IsBound = false.
🟠public XYZ Direction { get; }
Gets the unit direction vector from start to end point. Always normalised to length 1.0.
ClassMaterial
Inherits: Element
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents a Revit material definition including surface graphics, cut graphics, render appearance, physical properties, and thermal properties. Materials are referenced by CompoundStructure layers and by element geometry parameters.
🟢 Static Factory + Properties + Methods
🟣public static ElementId Create(Document doc, string name)
Creates a new material with the given name. Returns the ElementId of the new material. The material starts with default white colour and no patterns.
// Example — Create and configure a material
using (var tx = new Transaction(doc, "Material"))
{
    tx.Start();
    ElementId matId = Material.Create(doc, "Concrete - Exposed");
    Material  mat   = doc.GetElement(matId) as Material;

    mat.Color            = new Color(180, 175, 170);
    mat.Transparency     = 0;    // 0–100
    mat.Shininess        = 20;   // 0–128
    mat.SmoothnessFactor = 50;
    mat.MaterialClass    = "Concrete";

    // Set cut fill colour
    mat.CutForegroundPatternColor = new Color(100, 100, 100);
    tx.Commit();
}

// Find material by name
Material concrete = new FilteredElementCollector(doc)
    .OfClass(typeof(Material))
    .Cast<Material>()
    .FirstOrDefault(m => m.Name.Contains("Concrete"));
ClassOptions
Inherits: System.Object
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Controls how geometry is retrieved via Element.get_Geometry(). Three key settings: View (which view's visibility rules apply), ComputeReferences (needed for dimensioning), and IncludeNonVisibleObjects.
🟠 Properties
🟠public View View { get; set; }
The view whose visibility settings are applied during geometry retrieval. Pass null for model geometry without view filtering. Setting a view may return different geometry (e.g. symbolic representation in plan views).
🟠public bool ComputeReferences { get; set; }
When true, Face.Reference and Edge.Reference are populated, enabling dimensioning. Setting this to true is slower but required for any workflow involving NewDimension().
🟠public bool IncludeNonVisibleObjects { get; set; }
When true, geometry from invisible lines, reference planes and symbolic lines is included. Usually set to false for solid geometry extraction.
// Example — Options configurations
// For solid extraction (volume, area calculations)
var solidOpts = new Options
{
    ComputeReferences        = false,
    IncludeNonVisibleObjects = false
};

// For dimensioning (must have references)
var dimOpts = new Options
{
    ComputeReferences        = true,
    View                     = doc.ActiveView,
    IncludeNonVisibleObjects = false
};

// For symbolic geometry in plan view
var symOpts = new Options
{
    View                     = doc.ActiveView,
    IncludeNonVisibleObjects = true
};
ClassOverrideGraphicSettings
Inherits: APIObject
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Defines graphic overrides applied to elements or categories in a view. Used with View.SetElementOverrides() (per element) or ParameterFilterElement + View.SetFilterOverrides() (per rule). Controls colour, line weight, pattern, transparency, and halftone for projection and cut representations.
🟢 Key Methods
// Example — Full override workflow
var ogs = new OverrideGraphicSettings();

// Projection (visible faces) settings
ogs.SetProjectionLineColor(new Color(255, 0, 0));
ogs.SetProjectionLineWeight(6);        // 1–16
ogs.SetSurfaceTransparency(30);        // 0–100
ogs.SetSurfaceForegroundPatternColor(new Color(255, 200, 200));
ogs.SetIsHalftone(false);

// Cut (section cut) settings
ogs.SetCutLineColor(new Color(200, 0, 0));
ogs.SetCutLineWeight(8);
ogs.SetCutFillColor(new Color(255, 200, 200));

// Apply to specific elements
using (var tx = new Transaction(doc, "Override"))
{
    tx.Start();
    foreach (var wall in walls)
        doc.ActiveView.SetElementOverrides(wall.Id, ogs);
    tx.Commit();
}

// Clear overrides — pass a new empty OverrideGraphicSettings
using (var tx = new Transaction(doc, "Clear"))
{
    tx.Start();
    doc.ActiveView.SetElementOverrides(wall.Id,
        new OverrideGraphicSettings());  // empty = reset to default
    tx.Commit();
}
ClassParameter
Inherits: APIObject
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents a single parameter on an element. Provides typed access to the parameter's value (AsString, AsDouble, AsInteger, AsElementId) and metadata (StorageType, IsReadOnly, Definition). All writes require an open Transaction.
🟠 Key Properties
🟠public StorageType StorageType { get; }
Gets how the value is stored internally: String, Double, Integer, or ElementId. Always check StorageType before calling the read methods.
🟠public bool IsReadOnly { get; }
True if the parameter cannot be written. Computed parameters (like Host Area Computed) are always read-only. Always check before calling Set().
🟠public Definition Definition { get; }
Gets the definition of this parameter, including its name, parameter type, and group. Cast to InternalDefinition for built-in parameters or ExternalDefinition for shared parameters.
🟠public bool IsShared { get; }
True if this is a shared parameter (defined in a shared parameter file with a GUID). Shared parameters are accessible by GUID via elem.get_Parameter(Guid).
🟢 Read Methods
🟢public string AsString()
Returns the value as a string. Only valid when StorageType == StorageType.String. Returns null for other types.
🟢public double AsDouble()
Returns the value as a double in internal units (feet for lengths, sq.ft for areas etc.). Only valid when StorageType == StorageType.Double. Always convert with UnitUtils.
🟢public int AsInteger()
Returns the value as an integer. Valid for StorageType.Integer. Used for enums, Yes/No (1/0), and integer counts.
🟢public ElementId AsElementId()
Returns the value as an ElementId. Valid for StorageType.ElementId. Used for material parameters, level parameters, phase parameters etc.
🟢public string AsValueString()
Returns the value as a user-facing formatted string regardless of StorageType. For length doubles this returns "4500mm" rather than a raw number. Useful for display purposes only.
🟢 Write Methods
🟢public bool Set(string value) // also: Set(double), Set(int), Set(ElementId)
Sets the parameter value. Must match the parameter's StorageType — call the correct overload. Returns true if set successfully. Must be inside an open Transaction. Check IsReadOnly before calling.
// Example — Complete parameter read/write pattern
Element elem = ...;
Parameter p  = elem.get_Parameter(BuiltInParameter.ALL_MODEL_MARK);

if (p == null) return;

// Read based on StorageType
switch (p.StorageType)
{
    case StorageType.String:
        string s = p.AsString();                 break;
    case StorageType.Double:
        double mm = p.AsDouble() * 304.8;        break;
    case StorageType.Integer:
        int    n  = p.AsInteger();               break;
    case StorageType.ElementId:
        ElementId eid = p.AsElementId();         break;
}

// Write (inside a transaction)
using (var tx = new Transaction(doc, "Set Param"))
{
    tx.Start();
    if (!p.IsReadOnly)
    {
        bool ok = p.Set("W-001");   // string overload
        // p.Set(3.281);              // double overload (feet)
        // p.Set(1);                  // int overload
        // p.Set(new ElementId(x));   // ElementId overload
    }
    tx.Commit();
}
ClassPlane
Inherits: APIObject
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Represents an infinite 2D plane in 3D space, defined by an origin and a normal vector. Used to create SketchPlanes, to project points, and to define arc orientations. Plane is immutable — create a new one rather than modifying an existing one.
🟢 Static Factory Methods
🟣public static Plane CreateByNormalAndOrigin(XYZ normal, XYZ origin)
Creates a plane from a normal vector and an origin point. The most common factory method. The normal defines which direction is "up" for the plane.
// Example — Plane creation patterns
// XY plane at Z=0 (horizontal)
Plane xy = Plane.CreateByNormalAndOrigin(XYZ.BasisZ, XYZ.Zero);

// Vertical plane along X-axis
Plane xz = Plane.CreateByNormalAndOrigin(XYZ.BasisY, XYZ.Zero);

// Plane from 3 points
Plane p3 = Plane.CreateByThreePoints(
    new XYZ(0,0,0),
    new XYZ(1,0,0),
    new XYZ(0,1,0));

// Create a SketchPlane from a Plane (needed for family geometry)
using (var tx = new Transaction(doc, "SketchPlane"))
{
    tx.Start();
    SketchPlane sp = SketchPlane.Create(doc, xy);
    tx.Commit();
}
ClassReferenceIntersector
Inherits: APIObject
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Performs ray casting against element geometry in a 3D view. Given a ray (origin + direction), ReferenceIntersector finds all elements that the ray hits. Used for proximity detection, height finding, automatic pipe-to-floor opening creation, and sight-line analysis.
🟣 Constructors
🟣public ReferenceIntersector(View3D view)
Creates a ReferenceIntersector that tests against all geometry visible in the given 3D view. The view must be a View3D — the ray casting works in the 3D view's coordinate space.
// Example — Ray cast downward to find floor elevation
// Get or create a 3D view for ray casting

ClassTransaction
Inherits: APIObject · IDisposable
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Wraps a set of model changes into a single undo/redo step. Every write to the Revit model (creating elements, modifying parameters, deleting) MUST happen inside an open Transaction. Transactions must be Started, then Committed or RolledBack. Use the using() pattern for automatic disposal.
// Critical Rules1) Never start a Transaction inside an event handler that fires during a transaction. 2) Never call Revit API from a background thread — use IExternalEventHandler. 3) Always check tx.GetStatus() in critical workflows. 4) Use TransactionGroup to batch multiple Transactions into one undo step.
🟣 Constructors
🟣public Transaction(Document doc, string name)
Creates a Transaction for the given document. The name appears in the Undo history. The transaction is NOT yet started — call Start() before making changes.
ParameterTypeDescription
docDocumentThe document to create the transaction in.
namestringName shown in the Revit Undo/Redo history. Should be descriptive.
// Example — All Transaction patterns
// Pattern 1: Basic (using = auto dispose)
using (var tx = new Transaction(doc, "Create Wall"))
{
    tx.Start();
    // ... modify model ...
    tx.Commit();
}

// Pattern 2: With error handling
using (var tx = new Transaction(doc, "Safe Create"))
{
    tx.Start();
    try
    {
        // ... risky work ...
        tx.Commit();
    }
    catch (Exception ex)
    {
        tx.RollBack();
        TaskDialog.Show("Error", ex.Message);
    }
}

// Pattern 3: TransactionGroup — one undo step for multiple transactions
using (var tg = new TransactionGroup(doc, "Batch Update"))
{
    tg.Start();
    using (var tx1 = new Transaction(doc, "Step 1"))
    { tx1.Start(); /* work */ tx1.Commit(); }
    using (var tx2 = new Transaction(doc, "Step 2"))
    { tx2.Start(); /* work */ tx2.Commit(); }
    tg.Assimilate();   // merge into one undo step
}

// Pattern 4: SubTransaction (partial rollback inside a transaction)
using (var tx = new Transaction(doc, "Main"))
{
    tx.Start();
    using (var sub = new SubTransaction(doc))
    {
        sub.Start();
        try { /* risky work */ sub.Commit(); }
        catch { sub.RollBack(); }   // main tx continues
    }
    tx.Commit();
}

// Pattern 5: Suppress warnings
using (var tx = new Transaction(doc, "No Warnings"))
{
    FailureHandlingOptions opts = tx.GetFailureHandlingOptions();
    opts.SetFailuresPreprocessor(new SuppressWarningsPreprocessor());
    tx.SetFailureHandlingOptions(opts);
    tx.Start();
    /* work that might produce warnings */
    tx.Commit();
}

InterfaceIExternalEventHandler
Namespace: Autodesk.Revit.UI · Assembly: RevitAPIUI.dll
The bridge between background threads/WPF windows and the Revit API. Because the Revit API cannot be called from background threads, implement this interface to queue work that will be executed in the main Revit thread at the next idle cycle when ExternalEvent.Raise() is called.
// The Bridge PatternBackground thread → sets data on handler properties → calls ExternalEvent.Raise() → Revit calls Execute(uiapp) on the main thread → reads data → calls API. This is thread-safe because all API calls happen on the main thread.
🟢 Required Methods
🟢public void Execute(UIApplication app)
Called by Revit on the main thread when ExternalEvent.Raise() has been called. This runs in a valid Revit API context — you can start transactions, modify elements, etc. Use public properties on the handler to pass data from the background thread.
// Example — Complete IExternalEventHandler + ExternalEvent pattern
// Step 1: Implement the handler
public class SetMarkHandler : IExternalEventHandler
{
    // Thread-safe data transfer via properties
    public ElementId TargetId    { get; set; }
    public string    MarkValue   { get; set; }
    public bool      Success     { get; private set; }

    public void Execute(UIApplication app)
    {
        try
        {
            Document doc = app.ActiveUIDocument?.Document;
            if (doc == null) { Success = false; return; }

            using (var tx = new Transaction(doc, "Set Mark"))
            {
                tx.Start();
                Element   elem = doc.GetElement(TargetId);
                Parameter mark = elem?.get_Parameter(
                    BuiltInParameter.ALL_MODEL_MARK);
                if (mark != null && !mark.IsReadOnly)
                    mark.Set(MarkValue);
                tx.Commit();
                Success = true;
            }
        }
        catch { Success = false; }
    }

    public string GetName() => "Set Mark Handler";
}

// Step 2: Register in OnStartup
var handler = new SetMarkHandler();
var extEvent = ExternalEvent.Create(handler);

// Step 3: Raise from WPF button click
private void BtnApply_Click(object sender, RoutedEventArgs e)
{
    handler.TargetId  = selectedElementId;
    handler.MarkValue = txtMark.Text;

    ExternalEventRequest req = extEvent.Raise();

    switch (req)
    {
        case ExternalEventRequest.Accepted:
            lblStatus.Content = "Queued ✔"; break;
        case ExternalEventRequest.Pending:
            lblStatus.Content = "Previous request still pending"; break;
        default:
            lblStatus.Content = $"Request {req}"; break;
    }
}

// Example — Pre-save validator
private void OnDocumentSaving(object sender, DocumentSavingEventArgs e)
{
    Document doc = e.Document;

    // Check for duplicate marks before saving
    var marks = new FilteredElementCollector(doc)
        .OfClass(typeof(Wall))
        .Cast<Wall>()
        .Select(w => w.get_Parameter(BuiltInParameter.ALL_MODEL_MARK)?.AsString())
        .Where(m => !string.IsNullOrEmpty(m))
        .ToList();

    bool hasDuplicates = marks.Count != marks.Distinct().Count();
    if (hasDuplicates)
    {
        TaskDialog.Show("Save Blocked", "Duplicate wall marks found! Fix before saving.");
        e.Cancel();   // prevent save
    }
}

InterfaceIFailuresPreprocessor
Namespace: Autodesk.Revit.DB
Allows a command to intercept and handle transaction failures (warnings and errors) before Revit shows its default failure dialog. Implement to silently suppress warnings or handle specific error types programmatically.
🟢 Required Method
🟢public FailureProcessingResult PreprocessFailures(FailuresAccessor failuresAccessor)
Called when failures occur during transaction commit. Inspect failures, delete warnings, or return ProceedWithRollBack to abort. Return Continue to proceed with commit after handling warnings.
// Example — Suppress all warnings silently
public class SuppressAllWarnings : IFailuresPreprocessor
{
    public FailureProcessingResult PreprocessFailures(
        FailuresAccessor accessor)
    {
        foreach (var failure in accessor.GetFailureMessages())
        {
            if (failure.GetSeverity() == FailureSeverity.Warning)
                accessor.DeleteWarning(failure);
            else
                return FailureProcessingResult.ProceedWithRollBack;
        }
        return FailureProcessingResult.Continue;
    }
}

// Usage in a transaction
using (var tx = new Transaction(doc, "Safe Op"))
{
    FailureHandlingOptions fho = tx.GetFailureHandlingOptions();
    fho.SetFailuresPreprocessor(new SuppressAllWarnings());
    tx.SetFailureHandlingOptions(fho);
    tx.Start();
    /* ... */
    tx.Commit();
}
