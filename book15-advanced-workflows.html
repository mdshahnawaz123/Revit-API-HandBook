Autodesk.Revit.DB.ExternalService

Plug custom server implementations into Revit's extensible service framework — routing preferences, pipe sizing, structural load classification, and DirectContext3D rendering.

// What is ExternalService?Revit's ExternalService framework lets add-ins replace or extend built-in Revit services. For example, you can register a custom routing preference server to control how pipes route around obstacles, or a pipe sizing server to apply your own sizing rules. DirectContext3D (Book 14) also uses this framework.
InterfaceIExternalServer
Namespace: Autodesk.Revit.DB.ExternalService
The base interface for all external servers. Any class that plugs into Revit's service framework must implement IExternalServer (or a sub-interface like IDirectContext3DServer, IRoutingPreferenceServer). Provides the server's identity: GUID, vendor, name, and which service it belongs to.
🟢 Required Methods
// Example — Routing Preference Server (MEP pipe routing)
using Autodesk.Revit.DB.ExternalService;
using Autodesk.Revit.DB.Plumbing;

// ── Custom Routing Preference Server ──
// Controls how Revit routes pipes: which fittings to prefer,
// how to handle junctions, what sizes to use.

public class StudioRoutingPreferenceServer :
    IRoutingPreferenceServer  // sub-interface of IExternalServer
{
    private Guid _id = new Guid("11223344-AABB-CCDD-EEFF-001122334455");

    // IExternalServer required
    public Guid   GetServiceId()   =>
        ExternalServices.BuiltInExternalServices.RoutingPreferenceService;
    public Guid   GetServerId()    => _id;
    public string GetVendorId()    => "STUDIO";
    public string GetName()        => "Studio Routing Preferences";
    public string GetDescription() => "Custom pipe routing preference rules";
    public string GetApplicationId() => "StudioTools";

    // IRoutingPreferenceServer — return preferred fitting for a junction type
    public IList<RoutingCriteria> GetPreferences(
        RoutingConditions conditions)
    {
        var result = new List<RoutingCriteria>();

        // Prefer elbows over mitered connections
        result.Add(new RoutingCriteria
        {
            CriteriaName    = "ElbowPreference",
            PreferenceType  = RoutingPreferenceRuleGroupType.Elbows,
            PriorityLevel   = 1
        });

        return result;
    }
}

// ── Registration in OnStartup ──
SingleServerService svc = ExternalServiceRegistry
    .GetService(ExternalServices.BuiltInExternalServices.RoutingPreferenceService)
    as SingleServerService;

var server = new StudioRoutingPreferenceServer();
svc?.AddServer(server);
svc?.SetActiveServer(server.GetServerId());
ClassExternalServiceRegistry
Inherits: Static utility
Namespace: Autodesk.Revit.DB.ExternalService
Static registry for all Revit external services. Use GetService() to retrieve a service by its GUID, then add/remove/activate servers in that service.
🟡 BuiltInExternalServices — Key Service GUIDs
Service PropertyTypePurpose
DirectContext3DServiceMultiServerServiceCustom 3D rendering (IDirectContext3DServer) — Book 14
RoutingPreferenceServiceSingleServerServiceMEP pipe/duct routing fitting preferences
PipeSizingServiceSingleServerServiceCustom pipe sizing rules (velocity/pressure methods)
DuctSizingServiceSingleServerServiceCustom duct sizing rules
WireServiceSingleServerServiceElectrical wire calculation server
LoadClassificationServiceSingleServerServiceStructural load classification rules
AnalyticalSupportServiceSingleServerServiceAnalytical model support conditions
CableTrayRoutingPreferenceServiceSingleServerServiceCable tray fitting preferences
ConduitRoutingPreferenceServiceSingleServerServiceConduit fitting preferences
// Example — ExternalServiceRegistry patterns
// Get MultiServerService (e.g. DirectContext3D — many servers active)
MultiServerService multiSvc = ExternalServiceRegistry.GetService(
    ExternalServices.BuiltInExternalServices.DirectContext3DService)
    as MultiServerService;

multiSvc?.AddServer(myServer);
IList<Guid> active = multiSvc?.GetActiveServerIds() ?? new List<Guid>();
active.Add(myServer.GetServerId());
multiSvc?.SetActiveServers(active);

// Get SingleServerService (e.g. pipe sizing — only one server active)
SingleServerService singleSvc = ExternalServiceRegistry.GetService(
    ExternalServices.BuiltInExternalServices.PipeSizingService)
    as SingleServerService;

singleSvc?.AddServer(mySizingServer);
singleSvc?.SetActiveServer(mySizingServer.GetServerId());

// Remove a server on shutdown
multiSvc?.RemoveServer(myServer.GetServerId());

// List all registered servers in a service
foreach (var id in multiSvc?.GetServerIds() ?? new List<Guid>())
{
    IExternalServer srv = multiSvc.GetServer(id);
    System.Diagnostics.Debug.WriteLine(
        $"Server: {srv.GetName()} [{srv.GetVendorId()}]");
}

Curtain Walls — CurtainGrid · CurtainGridLine · Panel

Read and manipulate curtain wall grids, panels, and mullions programmatically.

ClassCurtainGrid
Inherits: APIObject
Namespace: Autodesk.Revit.DB · Accessed via Wall.CurtainGrid
Represents the grid system of a curtain wall — the system of U (horizontal) and V (vertical) grid lines that divide the wall into panels. Only available on Wall elements whose WallType.Kind == WallKind.Curtain. Provides access to all grid lines, panels (cells), and mullion layouts.
// Curtain Wall CheckAlways check wall.WallType.Kind == WallKind.Curtain before accessing wall.CurtainGrid. For non-curtain walls, CurtainGrid returns null. Modifying the grid (adding/removing lines) requires a Transaction.
🟠 Key Properties
🟠public ICollection<ElementId> GetUGridLineIds()
Returns the IDs of all U-direction (horizontal) grid lines. Cast each to CurtainGridLine.
🟠public ICollection<ElementId> GetVGridLineIds()
Returns the IDs of all V-direction (vertical) grid lines.
🟠public ICollection<ElementId> GetPanelIds()
Returns the IDs of all curtain panels (the infill elements — glazing, solid panels etc.) in this grid.
🟠public ICollection<ElementId> GetMullionIds()
Returns the IDs of all Mullion elements (the structural/decorative framing members between panels).
// Example — Complete CurtainGrid analysis
// Get all curtain walls
var curtainWalls = new FilteredElementCollector(doc)
    .OfClass(typeof(Wall)).Cast<Wall>()
    .Where(w => w.WallType.Kind == WallKind.Curtain)
    .ToList();

foreach (var cw in curtainWalls)
{
    CurtainGrid grid = cw.CurtainGrid;
    if (grid == null) continue;

    // Count grid lines and panels
    int uLines   = grid.GetUGridLineIds().Count;
    int vLines   = grid.GetVGridLineIds().Count;
    int panels   = grid.GetPanelIds().Count;
    int mullions = grid.GetMullionIds().Count;

    System.Diagnostics.Debug.WriteLine(
        $"Curtain Wall {cw.Id}: U={uLines} V={vLines} Panels={panels} Mullions={mullions}");

    // Analyse each panel
    foreach (ElementId panelId in grid.GetPanelIds())
    {
        Panel      panel    = doc.GetElement(panelId) as Panel;
        FamilySymbol sym  = panel?.get_Parameter(BuiltInParameter.ELEM_TYPE_PARAM)
                                 ?.AsValueString() is string typeName
                                 ? null : null;

        // Panel bounding box
        BoundingBoxXYZ bb = panel?.get_BoundingBox(null);
        if (bb != null)
        {
            double widthMm  = (bb.Max.X - bb.Min.X) * 304.8;
            double heightMm = (bb.Max.Z - bb.Min.Z) * 304.8;
            System.Diagnostics.Debug.WriteLine(
                $"  Panel: W={widthMm:F0}mm H={heightMm:F0}mm");
        }
    }

    // Add a U grid line at a specific elevation
    using (var tx = new Transaction(doc, "Add Grid Line"))
    {
        tx.Start();
        ElementId newLineId = grid.AddGridLine(false, 0.5, false);
        // AddGridLine(isUGrid, position, lockToNearest)
        // position 0.0 = bottom, 1.0 = top, 0.5 = midpoint
        tx.Commit();
    }

    // Remove an empty grid line
    ICollection<ElementId> uLineIds = grid.GetUGridLineIds();
    if (uLineIds.Count > 2)
    {
        using (var tx = new Transaction(doc, "Remove Grid Line"))
        {
            tx.Start();
            ElementId lineToRemove = uLineIds.Last();
            grid.RemoveGridLine(lineToRemove);
            tx.Commit();
        }
    }
}

// Change panel type for all empty panels
FamilySymbol solidPanelType = new FilteredElementCollector(doc)
    .OfClass(typeof(FamilySymbol))
    .OfCategory(BuiltInCategory.OST_CurtainWallPanels)
    .Cast<FamilySymbol>()
    .FirstOrDefault(s => s.Name.Contains("Solid"));

if (solidPanelType != null)
{
    using (var tx = new Transaction(doc, "Change Panels"))
    {
        tx.Start();
        foreach (var cw in curtainWalls)
        {
            foreach (ElementId pid in cw.CurtainGrid?.GetPanelIds()
                ?? new List<ElementId>())
            {
                Panel p = doc.GetElement(pid) as Panel;
                if (p != null)
                    p.ChangeTypeId(solidPanelType.Id);
            }
        }
        tx.Commit();
    }
}

Adaptive Components

Place and read adaptive family instances — families with flexible point-driven geometry used for complex parametric structures.

ClassAdaptiveComponentInstanceUtils
Inherits: Static utility
Namespace: Autodesk.Revit.DB
Static utility for placing, querying, and updating adaptive family instances. Adaptive families have "adaptive points" that you position in 3D space — the family geometry deforms to fit. Used for parametric panels, structural connections, and complex facades.
🟢 Key Static Methods
🟢public static bool IsAdaptiveComponentInstance(FamilyInstance instance)
Returns true if the given FamilyInstance is an adaptive component. Always call this before trying to access adaptive point methods — it prevents exceptions on regular families.
🟢public static IList<ElementId> GetInstancePlacementPointElementRefIds(FamilyInstance instance)
Returns the IDs of the ReferencePoint elements that define this instance's adaptive placement. The order matches the adaptive point indices in the family (point 1, 2, 3…).
🟢public static FamilyInstance PlaceAdaptiveComponentInstance(Document doc, FamilySymbol symbol)
Creates a new adaptive component instance without any placement points set. After calling this, use SetInstancePlacementPointElementRefIds() or move the ReferencePoint elements to position the adaptive instance.
// Example — Place and position a 4-point adaptive panel
// Find the adaptive family symbol
FamilySymbol adaptiveSym = new FilteredElementCollector(doc)
    .OfClass(typeof(FamilySymbol))
    .Cast<FamilySymbol>()
    .FirstOrDefault(s => AdaptiveComponentFamilyUtils
        .IsAdaptiveComponentFamily(s.Family));

if (adaptiveSym == null) return Result.Failed;

using (var tx = new Transaction(doc, "Place Adaptive"))
{
    tx.Start();

    if (!adaptiveSym.IsActive) adaptiveSym.Activate();

    // Place the instance (unpositioned)
    FamilyInstance instance =
        AdaptiveComponentInstanceUtils
            .PlaceAdaptiveComponentInstance(doc, adaptiveSym);

    // Get the adaptive placement point elements
    IList<ElementId> ptIds =
        AdaptiveComponentInstanceUtils
            .GetInstancePlacementPointElementRefIds(instance);

    // Position each adaptive point in 3D space
    XYZ[] positions = {
        new XYZ( 0,  0, 0),   // Point 1 — bottom-left
        new XYZ(10,  0, 0),   // Point 2 — bottom-right
        new XYZ(10, 10, 5),   // Point 3 — top-right (elevated)
        new XYZ( 0, 10, 3),   // Point 4 — top-left (elevated)
    };

    for (int i = 0; i < ptIds.Count && i < positions.Length; i++)
    {
        ReferencePoint pt = doc.GetElement(ptIds[i]) as ReferencePoint;
        if (pt == null) continue;
        // Move the reference point to the target position
        (pt.Location as LocationPoint).Point = positions[i];
    }

    tx.Commit();
}

// ── Read adaptive point positions ──
var adaptives = new FilteredElementCollector(doc)
    .OfClass(typeof(FamilyInstance)).Cast<FamilyInstance>()
    .Where(AdaptiveComponentInstanceUtils.IsAdaptiveComponentInstance)
    .ToList();

foreach (var inst in adaptives)
{
    IList<ElementId> ptIds =
        AdaptiveComponentInstanceUtils
            .GetInstancePlacementPointElementRefIds(inst);

    System.Diagnostics.Debug.WriteLine(
        $"Adaptive instance {inst.Id}: {ptIds.Count} points");

    foreach (var ptId in ptIds)
    {
        ReferencePoint rp = doc.GetElement(ptId) as ReferencePoint;
        if (rp == null) continue;
        XYZ pos = (rp.Location as LocationPoint)?.Point;
        System.Diagnostics.Debug.WriteLine(
            $"  Point: ({pos?.X * 304.8:F0}, {pos?.Y * 304.8:F0}, {pos?.Z * 304.8:F0})mm");
    }
}

ColorFillScheme · DesignOption

Create colour-fill room/area schemes programmatically and manage design options.

ClassColorFillScheme
Inherits: Element
Namespace: Autodesk.Revit.DB
Represents a colour fill scheme — a mapping from room/area parameter values to colours displayed in floor plan views. Each scheme has a category (Rooms, Areas, Spaces), a title parameter, and a list of ColorFillSchemeEntry objects defining the value-to-colour mapping.
🟢 Creation + Usage
// Example — Create department colour fill scheme for rooms
using (var tx = new Transaction(doc, "Color Fill Scheme"))
{
    tx.Start();

    // Create scheme for rooms by department
    ColorFillScheme scheme = ColorFillScheme.Create(
        doc,
        ColorFillType.Rooms,                             // category
        new ElementId(BuiltInParameter.ROOM_DEPARTMENT)); // title parameter

    scheme.Title = "Departments";

    // Add colour entries for each department value
    var deptColors = new Dictionary<string, (byte R, byte G, byte B)>
    {
        { "Office",      (100, 180, 240) },  // blue
        { "Meeting",     ( 80, 200, 130) },  // green
        { "Reception",   (240, 200,  80) },  // yellow
        { "Circulation", (200, 140,  80) },  // orange
        { "WC",          (180,  80, 180) },  // purple
        { "Storage",     (160, 160, 160) },  // grey
    };

    int idx = 0;
    foreach (var kvp in deptColors)
    {
        var entry = new ColorFillSchemeEntry(StorageType.String)
        {
            Color       = new Color(kvp.Value.R, kvp.Value.G, kvp.Value.B),
            Caption     = kvp.Key,
            IsVisible   = true,
            StorageValue = kvp.Key
        };
        scheme.SetEntry(idx++, entry);
    }

    // Apply scheme to active floor plan view
    ViewPlan floorPlan = doc.ActiveView as ViewPlan;
    if (floorPlan != null)
        floorPlan.SetColorFillSchemeId(
            BuiltInCategory.OST_Rooms, scheme.Id);

    tx.Commit();
}

// ── Read existing scheme entries ──
var existingScheme = new FilteredElementCollector(doc)
    .OfClass(typeof(ColorFillScheme))
    .Cast<ColorFillScheme>()
    .FirstOrDefault(s => s.Title == "Departments");

if (existingScheme != null)
{
    int count = existingScheme.GetEntryCount();
    for (int i = 0; i < count; i++)
    {
        ColorFillSchemeEntry e = existingScheme.GetEntry(i);
        System.Diagnostics.Debug.WriteLine(
            $"  {e.Caption}: RGB({e.Color.Red},{e.Color.Green},{e.Color.Blue})");
    }
}
ClassDesignOption
Inherits: Element
Namespace: Autodesk.Revit.DB
Represents a design option — an alternative design variant stored within the same Revit model. Design options are grouped into DesignOptionSets. Elements can be assigned to a specific design option so they only appear when that option is active. Used for presenting multiple design alternatives to clients.
🟢 Create + Manage Design Options
// Example — Create design option set + options + move elements
using (var tx = new Transaction(doc, "Design Options"))
{
    tx.Start();

    // ── Create DesignOptionSet ──
    DesignOptionSet optionSet = DesignOptionSet.Create(doc);
    optionSet.Name = "Facade Design";

    // ── Create options within the set ──
    // Note: One option is automatically created as primary
    // Use DesignOption.Create() to add alternates
    DesignOption optA = DesignOption.Create(doc, optionSet.Id);
    optA.Name = "Option A - Glass";

    DesignOption optB = DesignOption.Create(doc, optionSet.Id);
    optB.Name = "Option B - Solid";

    tx.Commit();
}

// ── Move elements to a design option ──
DesignOption targetOption = new FilteredElementCollector(doc)
    .OfClass(typeof(DesignOption)).Cast<DesignOption>()
    .FirstOrDefault(o => o.Name == "Option A - Glass");

if (targetOption != null)
{
    using (var tx = new Transaction(doc, "Assign to Option"))
    {
        tx.Start();
        // Assign selected walls to Option A
        foreach (var wall in selectedWalls)
        {
            Parameter dp = wall.get_Parameter(BuiltInParameter.DESIGN_OPTION_ID);
            dp?.Set(targetOption.Id);
        }
        tx.Commit();
    }
}

// ── Read which option an element belongs to ──
foreach (var wall in walls)
{
    ElementId doId = wall.get_Parameter(BuiltInParameter.DESIGN_OPTION_ID)
                          ?.AsElementId();
    if (doId != null && doId != ElementId.InvalidElementId)
    {
        DesignOption opt = doc.GetElement(doId) as DesignOption;
        System.Diagnostics.Debug.WriteLine(
            $"Wall {wall.Id} in: {opt?.Name ?? "Main Model"}");
    }
}

// ── List all design option sets and options ──
var optSets = new FilteredElementCollector(doc)
    .OfClass(typeof(DesignOptionSet)).Cast<DesignOptionSet>();
foreach (var os in optSets)
{
    System.Diagnostics.Debug.WriteLine($"Set: {os.Name}");
    foreach (ElementId doId in os.GetDesignOptionIds())
    {
        DesignOption opt = doc.GetElement(doId) as DesignOption;
        System.Diagnostics.Debug.WriteLine(
            $"  Option: {opt?.Name} Primary={opt?.IsPrimary}");
    }
}

AssemblyInstance · AssemblyViewUtils

Group elements into assemblies (prefabrication packages) and automatically generate assembly views and sheets.

ClassAssemblyInstance
Inherits: Element
Namespace: Autodesk.Revit.DB
Represents an assembly — a named group of elements treated as a fabrication unit. Assemblies can have dedicated views, sheets, and assembly codes. Used for precast concrete, steel prefabrication, and MEP coordination packages.
🟢 Creation + AssemblyViewUtils
// Example — Create assembly from structural beam + generate views + sheet
// Collect elements for the assembly (a beam and its connections)
var beamIds = new FilteredElementCollector(doc)
    .OfCategory(BuiltInCategory.OST_StructuralFraming)
    .WhereElementIsNotElementType()
    .ToElementIds()
    .Take(3).ToList();  // first 3 beams for demo

AssemblyInstance assembly;

using (var tx = new Transaction(doc, "Create Assembly"))
{
    tx.Start();

    // Validate elements can form an assembly
    if (!AssemblyInstance.AreElementsValidForAssembly(doc, beamIds, ElementId.InvalidElementId))
    {
        tx.RollBack();
        TaskDialog.Show("Error", "Elements cannot form an assembly");
        return Result.Failed;
    }

    // Create the assembly
    assembly = AssemblyInstance.Create(doc, beamIds, beamIds[0]);
    // Second param = naming category element ID

    assembly.AssemblyTypeName = "Beam Assembly Type A";

    // Set assembly code (for schedules/BOQ)
    assembly.get_Parameter(BuiltInParameter.ASSEMBLY_CODE)
        ?.Set("BA-001");

    tx.Commit();
}

// ── Generate assembly views using AssemblyViewUtils ──
using (var tx = new Transaction(doc, "Assembly Views"))
{
    tx.Start();

    // Get 3D view type for assembly views
    ElementId v3dTypeId = new FilteredElementCollector(doc)
        .OfClass(typeof(ViewFamilyType)).Cast<ViewFamilyType>()
        .First(v => v.ViewFamily == ViewFamily.ThreeD).Id;

    // Create a 3D view of the assembly
    View3D assemblyView3D =
        AssemblyViewUtils.Create3DOrthographic(doc, assembly.Id, v3dTypeId);
    assemblyView3D.Name = $"3D - {assembly.AssemblyTypeName}";

    // Create detail views (front, back, left, right, top, bottom)
    ElementId sectionTypeId = new FilteredElementCollector(doc)
        .OfClass(typeof(ViewFamilyType)).Cast<ViewFamilyType>()
        .First(v => v.ViewFamily == ViewFamily.Section).Id;

    ViewSection frontView =
        AssemblyViewUtils.CreateDetailSection(
            doc, assembly.Id, sectionTypeId,
            AssemblyDetailViewOrientation.ElevationFront,
            50);   // view scale 1:50

    ViewSection sideView =
        AssemblyViewUtils.CreateDetailSection(
            doc, assembly.Id, sectionTypeId,
            AssemblyDetailViewOrientation.ElevationLeft, 50);

    // Create an assembly sheet with all views placed
    FamilySymbol titleBlock = new FilteredElementCollector(doc)
        .OfCategory(BuiltInCategory.OST_TitleBlocks)
        .OfClass(typeof(FamilySymbol))
        .Cast<FamilySymbol>().First();

    ViewSheet sheet =
        AssemblyViewUtils.CreateSheet(doc, assembly.Id, titleBlock.Id);
    sheet.SheetNumber = "FA-001";
    sheet.Name        = $"Fabrication - {assembly.AssemblyTypeName}";

    // Automatically place the views on the sheet
    AssemblyViewUtils.PopulateSheet(sheet, assembly.Id);

    tx.Commit();
}

PointClouds — Scan to BIM

Access point cloud data, query scan points, and work with RCP/RCS files for scan-to-BIM workflows.

ClassPointCloudInstance
Inherits: Element
Namespace: Autodesk.Revit.DB.PointClouds
Represents a linked point cloud file (RCP/RCS/PCG) placed in the Revit model. Provides access to point data through filters, the transform from point cloud space to model space, and cloud metadata. The actual point data is retrieved via GetPoints() with an optional PointCloudFilter.
🟢 Key Methods + Usage
🟢public PointCollection GetPoints(PointCloudFilter filter, double averageDistance, int numPoints)
Returns a collection of CloudPoint objects matching the filter. averageDistance controls the average spacing between returned points (larger = fewer points, faster). numPoints limits the maximum number returned.
ParameterTypeDescription
filterPointCloudFilterSpatial or colour filter. Pass null for no filter.
averageDistancedoubleDesired average distance between returned points in feet. Use 0 for max density.
numPointsintMaximum number of points to return. Use int.MaxValue for all.
Returns: PointCollection — Collection of CloudPoint (X, Y, Z, Color) values.
// Example — Query point cloud within a bounding region
using Autodesk.Revit.DB.PointClouds;

// Get all loaded point cloud instances
var pcs = new FilteredElementCollector(doc)
    .OfClass(typeof(PointCloudInstance))
    .Cast<PointCloudInstance>()
    .ToList();

foreach (var pc in pcs)
{
    // Get the point cloud's transform (cloud space → model space)
    Transform xf = pc.GetTransform();

    // Create a bounding box filter for a specific room area
    BoundingBoxXYZ searchBB = new BoundingBoxXYZ();
    searchBB.Min = new XYZ(0, 0, 0);
    searchBB.Max = new XYZ(20, 20, 5);

    // Create axis-aligned bounding box filter in model coordinates
    PointCloudFilter filter = PointCloudFilterFactory
        .CreateMultiPlaneFilter(
            new List<Plane>
            {
                Plane.CreateByNormalAndOrigin(XYZ.BasisX, searchBB.Min),
                Plane.CreateByNormalAndOrigin(-XYZ.BasisX, searchBB.Max),
                Plane.CreateByNormalAndOrigin(XYZ.BasisY, searchBB.Min),
                Plane.CreateByNormalAndOrigin(-XYZ.BasisY, searchBB.Max),
                Plane.CreateByNormalAndOrigin(XYZ.BasisZ, searchBB.Min),
                Plane.CreateByNormalAndOrigin(-XYZ.BasisZ, searchBB.Max)
            });

    // Query points at ~50mm spacing, max 10,000 points

Pattern 2Complete Clash Detection Workflow
🟢 Structural vs MEP Boolean Intersection Clash Detection
// Example — Find pipe-to-beam clashes using BooleanOperationsUtils
// Helper: get first valid solid from element
Solid GetSolid(Element e)
{
    var opts = new Options { ComputeReferences = false };
    foreach (var obj in e.get_Geometry(opts))
    {
        if (obj is Solid s && s.Volume > 1e-9) return s;
        if (obj is GeometryInstance gi)
            foreach (var o in gi.GetInstanceGeometry())
                if (o is Solid s2 && s2.Volume > 1e-9) return s2;
    }
    return null;
}

// Collect structural beams and pipes
var beams = new FilteredElementCollector(doc)
    .OfCategory(BuiltInCategory.OST_StructuralFraming)
    .WhereElementIsNotElementType().ToList();
var pipes = new FilteredElementCollector(doc)
    .OfClass(typeof(Autodesk.Revit.DB.Plumbing.Pipe))
    .ToList();

var clashes = new List<(ElementId beamId, ElementId pipeId, double volumeM3)>();

foreach (var beam in beams)
{
    Solid beamSolid = GetSolid(beam);
    if (beamSolid == null) continue;

    // Pre-filter: only check pipes whose bounding box overlaps
    BoundingBoxXYZ beamBB = beam.get_BoundingBox(null);
    if (beamBB == null) continue;

    var outline    = new Outline(beamBB.Min, beamBB.Max);
    var bboxFilter = new BoundingBoxIntersectsFilter(outline);
    var nearbyPipes = pipes
        .Where(p => {
            var pb = p.get_BoundingBox(null);
            return pb != null
                && pb.Max.X >= beamBB.Min.X && pb.Min.X <= beamBB.Max.X
                && pb.Max.Y >= beamBB.Min.Y && pb.Min.Y <= beamBB.Max.Y
                && pb.Max.Z >= beamBB.Min.Z && pb.Min.Z <= beamBB.Max.Z;
        }).ToList();

    foreach (var pipe in nearbyPipes)
    {
        Solid pipeSolid = GetSolid(pipe);
        if (pipeSolid == null) continue;

        try
        {
            Solid intersection = BooleanOperationsUtils
                .ExecuteBooleanOperation(beamSolid, pipeSolid,
                    BooleanOperationsType.Intersect);

            if (intersection?.Volume > 1e-9)
            {
                double vol = intersection.Volume * 0.0283168;  // ft³→m³
                clashes.Add((beam.Id, pipe.Id, vol));
            }
        }
        catch { /* incompatible geometry — skip */ }
    }
}

// Report results
TaskDialog.Show("Clash Report",
    $"Found {clashes.Count} clashes\n" +
    string.Join("\n",
        clashes.Take(10).Select(c =>
            $"Beam {c.beamId} ↔ Pipe {c.pipeId}: {c.volumeM3 * 1e6:F1}cm³")));
