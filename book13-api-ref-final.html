Autodesk.Revit.ApplicationServices

The Application and ControlledApplication classes — top-level Revit session objects.

ClassApplication
Inherits: APIObject
Namespace: Autodesk.Revit.ApplicationServices · Assembly: RevitAPI.dll
Represents the Revit application session. Provides access to application-level settings, open documents, product version, username, and the ability to open/create documents. Accessed from IExternalCommand via commandData.Application.Application, or from IExternalApplication via ControlledApplication.
🟠 Key Properties
🟠public string VersionName { get; }
Gets the full version name string, e.g. "Autodesk Revit 2024". Use for logging and version-specific feature branching.
🟠public string VersionNumber { get; }
Gets the version number string, e.g. "2024". Parse to int for numeric comparisons.
🟠public string VersionBuild { get; }
Gets the full build number string (e.g. "24.0.1.0"). Useful for logging exact build information in bug reports.
🟠public string Username { get; }
Gets the current Revit username (from Autodesk account or network login). Used for worksharing ownership tracking and audit logs.
🟠public DocumentSet Documents { get; }
Gets all currently open documents (project and family files). Iterate to find a specific document or to perform batch operations across all open files.
🟠public ProductType Product { get; }
Gets the Revit product type: Revit, RevitLT, MEP, Structure, or Architecture. Used for feature availability checks.
🟠public LanguageType Language { get; }
Gets the current Revit UI language (English_USA, English_GB, German, French, Arabic, Chinese_Simplified, etc.). Use for parameter name lookups and localised text.
🟢 Key Methods
🟢public Document OpenDocumentFile(string filename)
Opens a Revit file (.rvt, .rfa, .rte) by file path and returns the Document. The file is opened as a regular project — not workshared. Use for batch processing multiple files.
🟢public Document OpenDocumentFile(ModelPath modelPath, OpenOptions openOptions)
Opens a document from a ModelPath (supports Revit Server and cloud models). OpenOptions allows setting audit, detach from central, and workset loading options.
🟢public Document NewFamilyDocument(string templateFileName)
Creates a new empty family document from the specified template (.rft file). The template determines the family category, reference planes, and coordinate system. Used when programmatically building new families from scratch.

ClassControlledApplication
Inherits: APIObject
Namespace: Autodesk.Revit.ApplicationServices
A restricted view of the Application accessible in IExternalApplication.OnStartup(). Unlike Application, ControlledApplication does not allow document-level operations during startup. Used primarily for subscribing to application-level events and reading version/language info.
🟠 Key Properties + Event Subscription
🟠public string VersionName { get; }
Same as Application.VersionName — full version string.
// Example — ControlledApplication in OnStartup
public Result OnStartup(UIControlledApplication app)
{
    ControlledApplication cApp = app.ControlledApplication;

    // Subscribe to all document lifecycle events
    cApp.DocumentOpened     += OnDocOpened;
    cApp.DocumentCreated    += OnDocCreated;
    cApp.DocumentSaving     += OnDocSaving;
    cApp.DocumentSaved      += OnDocSaved;
    cApp.DocumentClosing    += OnDocClosing;
    cApp.DocumentClosed     += OnDocClosed;
    cApp.DocumentChanged    += OnDocChanged;
    cApp.DocumentPrinting   += OnDocPrinting;
    cApp.DocumentSynchronizingWithCentral += OnSyncing;
    cApp.DocumentSynchronizedWithCentral  += OnSynced;

    // UI events on UIControlledApplication
    app.ViewActivated    += OnViewActivated;
    app.Idling           += OnIdling;
    app.ApplicationInitialized += OnAppInit;

    System.Diagnostics.Debug.WriteLine(
        $"Plugin loaded for Revit {cApp.VersionName} | User: {cApp.Username}");

    return Result.Succeeded;
}

public Result OnShutdown(UIControlledApplication app)
{
    // !! Always unsubscribe ALL events here !!
    app.ControlledApplication.DocumentOpened  -= OnDocOpened;
    app.ControlledApplication.DocumentChanged -= OnDocChanged;
    app.ControlledApplication.DocumentSaving  -= OnDocSaving;
    app.ControlledApplication.DocumentClosing -= OnDocClosing;
    app.ViewActivated                          -= OnViewActivated;
    app.Idling                                 -= OnIdling;
    return Result.Succeeded;
}

Autodesk.Revit.Exceptions

All Revit-specific exception types — when each is thrown and how to handle it.

ReferenceAll Exception Types
Namespace: Autodesk.Revit.Exceptions · Assembly: RevitAPI.dll
Revit defines its own exception hierarchy separate from System.Exception. All Revit exceptions inherit from RevitExceptions. Catch specific types for fine-grained error handling; catch the base RevitException for generic Revit errors.
🟡 Exception Types Reference
ExceptionThrown WhenHow to Handle
InvalidOperationExceptionAPI call made at wrong time (e.g. starting Transaction inside event, calling API from background thread)Check context — use ExternalEvent for background thread calls
ArgumentExceptionArgument value is invalid (null curve, degenerate geometry, etc.)Validate inputs before API call
ArgumentNullExceptionRequired argument is nullNull-check all inputs
ApplicationExceptionGeneral Revit application-level errorLog and report to user
OperationCanceledExceptionUser pressed Escape during PickObject, PickPoint, or PickObjectsAlways catch this after Selection.PickObject() calls
ExternalApplicationExceptionError in an external application's startup/shutdownCheck OnStartup/OnShutdown for exceptions
ForbiddenForDynamicUpdateExceptionIllegal operation inside IUpdater.Execute() (e.g. starting new Transaction)Modify elements directly inside Execute() — no new Transaction
RegenerationFailedExceptionModel regeneration failed after an operationRollBack the transaction; simplify the operation
InternalExceptionInternal Revit error — usually geometry or corrupted elementSkip element or rollback; log element ID
FileNotFoundExceptionFamily .rfa or template file not foundValidate file path before calling LoadFamily/OpenDocumentFile
FileAccessExceptionFile is locked, read-only, or permission deniedCheck file permissions; warn user
DocumentValidationExceptionDocument is in an invalid state for the operationCheck doc.IsReadOnly, doc.IsModified
// Example — Comprehensive exception handling pattern
public Result Execute(ExternalCommandData cd, ref string msg, ElementSet els)
{
    try
    {
        Document doc = cd.Application.ActiveUIDocument?.Document;
        if (doc == null) { msg = "No active document"; return Result.Failed; }

        // Selection — catch Escape (OperationCanceledException)
        Reference picked;
        try
        {
            picked = cd.Application.ActiveUIDocument.Selection
                .PickObject(ObjectType.Element, "Select element");
        }
        catch (Autodesk.Revit.Exceptions.OperationCanceledException)
        {
            return Result.Cancelled;   // user pressed Escape
        }

        using (var tx = new Transaction(doc, "Operation"))
        {
            tx.Start();
            try
            {
                // ... work ...
                tx.Commit();
            }
            catch (Autodesk.Revit.Exceptions.InvalidOperationException ex)
            {
                tx.RollBack();
                msg = $"Invalid operation: {ex.Message}";
                return Result.Failed;
            }
            catch (Autodesk.Revit.Exceptions.RegenerationFailedException)
            {
                tx.RollBack();
                msg = "Geometry regeneration failed — operation rolled back.";
                return Result.Failed;
            }
        }
        return Result.Succeeded;
    }
    catch (Exception ex)
    {
        // Catch-all for unexpected errors
        msg = $"Unexpected error: {ex.Message}";
        System.Diagnostics.Debug.WriteLine(ex.StackTrace);
        return Result.Failed;
    }
}

Namespace: Autodesk.Revit.UI.Selection
Controls what the user can select during a PickObject() or PickObjects() call. AllowElement() returns true for elements that should be selectable; AllowReference() filters sub-element picks (faces, edges).

// Example — All ISelectionFilter patterns
// ── Filter 1: Category-based ──
public class WallFilter : ISelectionFilter
{
    public bool AllowElement(Element e) => e is Wall;
    public bool AllowReference(Reference r, XYZ p) => true;
}

// ── Filter 2: Multiple categories ──
public class StructuralFilter : ISelectionFilter
{
    private static readonly HashSet<BuiltInCategory> _cats =
        new HashSet<BuiltInCategory>
        {
            BuiltInCategory.OST_StructuralColumns,
            BuiltInCategory.OST_StructuralFraming,
            BuiltInCategory.OST_StructuralFoundation
        };
    public bool AllowElement(Element e)
        => e.Category != null &&
           _cats.Contains((BuiltInCategory)e.Category.Id.IntegerValue);
    public bool AllowReference(Reference r, XYZ p) => true;
}

// ── Filter 3: Face-only (for dimensioning) ──
public class WallFaceFilter : ISelectionFilter
{
    public bool AllowElement(Element e) => e is Wall;
    public bool AllowReference(Reference r, XYZ p)
    {
        // Only allow face references (not edge or point references)
        return r.ElementReferenceType == ElementReferenceType.REFERENCE_TYPE_SURFACE;
    }
}

// ── Filter 4: Custom predicate ──
public class PredicateFilter : ISelectionFilter
{
    private readonly Func<Element, bool> _predicate;
    public PredicateFilter(Func<Element, bool> pred) => _predicate = pred;
    public bool AllowElement(Element e) => _predicate(e);
    public bool AllowReference(Reference r, XYZ p) => true;
}

// ── Usage ──
var sel = uidoc.Selection;

// Pick one wall only
try
{
    Reference r = sel.PickObject(ObjectType.Element,
        new WallFilter(), "Pick a wall");
    Wall wall = doc.GetElement(r) as Wall;
}
catch (Autodesk.Revit.Exceptions.OperationCanceledException) { }

// Pick multiple using custom predicate
var refs = sel.PickObjects(ObjectType.Element,
    new PredicateFilter(e => e.is FamilyInstance), "Pick family instances");

// Pick a face
Reference faceRef = sel.PickObject(ObjectType.Face,
    new WallFaceFilter(), "Click a wall face");

// Pick linked element
Reference linkRef = sel.PickObject(ObjectType.LinkedElement,
    "Pick element in linked model");
RevitLinkInstance link = doc.GetElement(linkRef.ElementId) as RevitLinkInstance;
Document          lDoc = link?.GetLinkDocument();
ElementId          lId  = linkRef.LinkedElementId;
Element            lElem = lDoc?.GetElement(lId);

Production-Grade Add-In Patterns

Do's and don'ts for every major API area — learned from real-world production add-ins.

⚙️ Transactions

// BP-01
Always use the using() pattern for Transactions
The using() pattern guarantees the Transaction is disposed even if an exception occurs, preventing document corruption from open/uncommitted transactions.
✓ DO
using (var tx = new Transaction(doc, "Op"))
{
    tx.Start();
    // work
    tx.Commit();
}
✗ DON'T
var tx = new Transaction(doc, "Op");
tx.Start();
// work — no using, no guarantee of commit/rollback!
tx.Commit();
tx.Dispose();  // easy to forget
// BP-02
One Transaction for many changes — not one per element
Each Transaction.Commit() triggers a full model regeneration. Creating 100 walls in 100 separate transactions is ~100× slower than creating them all in one transaction.
✓ DO — one transaction
using (var tx = new Transaction(doc, "Batch"))
{
    tx.Start();
    foreach (var item in items)
        ProcessItem(doc, item);  // all inside 1 tx
    tx.Commit();
}
✗ DON'T — one per item
foreach (var item in items)
{
    using (var tx = new Transaction(doc, "Item"))
    {
        tx.Start();
        ProcessItem(doc, item);
        tx.Commit();   // n regenerations!
    }
}
// BP-03
Use TransactionGroup to create a single Undo step
When your operation needs multiple sub-transactions (e.g. create elements, then modify them), wrap them in a TransactionGroup.Assimilate() to give the user one undo step instead of many.

⚡ Performance

// BP-04
Use OfClass() before OfCategory() — always chain quick filters first
OfClass() is the fastest filter. OfCategory() is slower. LINQ .Where() is slowest. Always apply them in order: OfClass → OfCategory → WherePasses(nativeFilter) → LINQ last.
✓ DO — fast chain
var doors = new FilteredElementCollector(doc)
    .OfClass(typeof(FamilyInstance))  // fast first
    .OfCategory(BuiltInCategory.OST_Doors)
    .Cast<FamilyInstance>()
    .Where(fi => fi.Host != null)  // LINQ last
    .ToList();
✗ DON'T — LINQ first
var doors = new FilteredElementCollector(doc)
    .ToElements()      // loads ALL elements!
    .Where(e => e is FamilyInstance)
    .Where(e => e.Category?.Id.IntegerValue
        == (int)BuiltInCategory.OST_Doors)
    .ToList();
// BP-05
Use ToElementIds() when you only need IDs
ToElementIds() is significantly faster and uses less memory than ToElements() because it doesn't load full element objects. Use when you're only doing selection, deletion, or external ID storage.
✓ DO — just IDs needed
// Select all walls without loading objects
var wallIds = new FilteredElementCollector(doc)
    .OfClass(typeof(Wall))
    .ToElementIds();
sel.SetElementIds(wallIds);
✗ DON'T — unnecessary objects
var walls = new FilteredElementCollector(doc)
    .OfClass(typeof(Wall))
    .ToElements();  // loads all objects!
var ids = walls.Select(w => w.Id).ToList();
sel.SetElementIds(ids);
// BP-06
Never call get_Geometry() with ComputeReferences=true unless you need references
ComputeReferences=true can be 2–5× slower than false. Only set it when you specifically need Face.Reference or Edge.Reference for dimensioning. For volume/area calculations, use ComputeReferences=false.
// BP-07
Cache document-level data on DocumentOpened — don't re-query every command
Querying all levels, all wall types, all family symbols on every command execution is wasteful. Cache these in a static dictionary on DocumentOpened, invalidate on DocumentChanged.

📡 Events

// BP-08
Always unsubscribe events in OnShutdown()
Failing to unsubscribe event handlers is a memory leak — each reference prevents garbage collection of the subscribing object and can cause Revit to crash on close.
✓ DO
public Result OnShutdown(UIControlledApplication app)
{
    app.ControlledApplication.DocumentOpened  -= OnOpened;
    app.ControlledApplication.DocumentChanged -= OnChanged;
    app.ViewActivated                          -= OnView;
    _externalEvent?.Dispose();
    return Result.Succeeded;
}
✗ DON'T
public Result OnShutdown(UIControlledApplication app)
{
    // Forgetting to unsubscribe = memory leaks + crashes
    return Result.Succeeded;
}
// BP-09
DocumentChanged is read-only — use IUpdater for write reactions
DocumentChanged fires after every commit in a READ-ONLY context. Any attempt to modify the model here throws InvalidOperationException. Use IUpdater (DMU) for reactions that need to write to the model, or queue writes via ExternalEvent.Raise().
// BP-10
Prefer ExternalEvent over Idling for modeless dialogs
Idling fires constantly even when nothing is happening, wasting CPU. ExternalEvent.Raise() fires only when explicitly requested and doesn't consume resources when idle. Use e.SetRaiseWithoutDelay(false) if you must use Idling.

🔀 Threading

// BP-11
NEVER call Revit API from a background thread
The Revit API is single-threaded. Calling any API method from Task.Run(), Thread, BackgroundWorker, or Timer will throw InvalidOperationException or corrupt the model. Always use IExternalEventHandler as the bridge.
✓ DO — ExternalEvent bridge
Task.Run(() =>
{
    // Heavy compute on background thread
    var result = HeavyCalculation();

    // Pass result to handler
    handler.Result = result;

    // Bridge back to main thread
    externalEvent.Raise();
});
✗ DON'T — direct API call
Task.Run(() =>
{
    // CRASH — cannot call API from here!
    var walls = new FilteredElementCollector(doc)
        .OfClass(typeof(Wall)).ToList();
});
// BP-12
Use ConcurrentDictionary for data shared between threads
If background threads need to write data that the ExternalEventHandler will read, use thread-safe collections (ConcurrentDictionary, ConcurrentQueue) or volatile fields with Interlocked operations.

🛡️ Error Handling

// BP-13
Always catch OperationCanceledException after PickObject
When the user presses Escape during a PickObject() call, Revit throws OperationCanceledException. If uncaught, this propagates as Result.Failed and confuses users. Always catch it and return Result.Cancelled.
// BP-14
Use IFailuresPreprocessor to suppress harmless warnings
Some operations (like creating walls near each other) produce non-critical warnings that interrupt your batch process. Use IFailuresPreprocessor to suppress specific warning types while still catching real errors.
// BP-15
Check IsReadOnly before calling Parameter.Set()
Many parameters are read-only (computed, locked, or instance parameters accessed from type). Calling Set() on a read-only parameter throws ArgumentException. Always check p.IsReadOnly before writing.
// BP-16
Null-check every API return value
Many API methods return null for valid reasons (element not found, parameter not present, no active view, etc.). A NullReferenceException inside a transaction can leave the model in a corrupted state. Defensive null-checking is essential.
✓ DO — defensive
Parameter p = elem?.get_Parameter(bip);
if (p == null || p.IsReadOnly) return;
double val = p.AsDouble();
if (val == null) return;
✗ DON'T — unsafe
// NullReferenceException if no param!
double val = elem
    .get_Parameter(bip)
    .AsDouble();

    catch { tx.RollBack(); throw; }
}

// 14. Extract solids from element
IEnumerable<Solid> GetSolids(Element e)
{
    var opts = new Options { ComputeReferences = false };
    foreach (var obj in e.get_Geometry(opts))
    {
        if (obj is Solid s && s.Volume > 1e-9) yield return s;
        if (obj is GeometryInstance gi)
            foreach (var o in gi.GetInstanceGeometry())
                if (o is Solid s2 && s2.Volume > 1e-9) yield return s2;
    }
}

// 17. Dismiss all warnings preprocessor
class SuppressWarningsPreprocessor : IFailuresPreprocessor
{
    public FailureProcessingResult PreprocessFailures(FailuresAccessor a)
    {
        foreach (var f in a.GetFailureMessages())
            if (f.GetSeverity() == FailureSeverity.Warning)
                a.DeleteWarning(f);
        return FailureProcessingResult.Continue;
    }
}
