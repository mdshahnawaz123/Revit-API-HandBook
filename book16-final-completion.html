ElementTransformUtils

Copy, move, rotate, and mirror elements — within the same document or across documents. The most-used utility class in production add-ins.

ClassElementTransformUtils
Inherits: Static utility
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Static utility class for all element transform operations: move, rotate, mirror, and copy (within and across documents). All methods require an open Transaction. CopyElements across documents is one of the most powerful API features — it preserves type information and hosted elements.
// Copy vs DuplicateUse ElementTransformUtils.CopyElements() for copying with position offsets. Use doc.GetElement(id).Clone() or duplicate-specific methods for types. Cross-document copy preserves all parameter values, hosted elements, and materials.
🟢 Complete Method Reference
🟢public static void MoveElement(Document doc, ElementId elementId, XYZ translation)
Moves a single element by the given translation vector. Moves the element's location curve or point by the vector — same as dragging in the UI. Hosted elements (doors in walls) move with their host when the host moves.
ParameterTypeDescription
docDocumentThe document containing the element.
elementIdElementIdID of the element to move.
translationXYZTranslation vector in internal units (feet). E.g. new XYZ(0,0,1) moves 1 foot upward.
// Example — Move, Rotate, Mirror
Wall      wall  = ...;
Document  doc   = wall.Document;

using (var tx = new Transaction(doc, "Transform"))
{
    tx.Start();

    // ── Move element 2000mm in Y direction ──
    double moveFt = UnitUtils.ConvertToInternalUnits(2000, UnitTypeId.Millimeters);
    ElementTransformUtils.MoveElement(doc, wall.Id, new XYZ(0, moveFt, 0));

    // ── Move multiple elements ──
    var idsToMove = walls.Select(w => w.Id).ToList();
    ElementTransformUtils.MoveElements(doc, idsToMove, new XYZ(0, moveFt, 0));

    // ── Rotate element around Z axis at its midpoint ──
    LocationCurve lc    = wall.Location as LocationCurve;
    XYZ           mid   = lc.Curve.Evaluate(0.5, true);
    Line          axis  = Line.CreateUnbound(mid, XYZ.BasisZ);
    double        angle = Math.PI / 2;  // 90 degrees

    ElementTransformUtils.RotateElement(doc, wall.Id, axis, angle);

    // ── Mirror element across the YZ plane ──
    Plane mirrorPlane = Plane.CreateByNormalAndOrigin(
        XYZ.BasisX, XYZ.Zero);  // mirror across X=0

    ElementTransformUtils.MirrorElement(doc, wall.Id, mirrorPlane);

    // ── Mirror with copy (original stays, mirrored copy created) ──
    ICollection<ElementId> mirrored =
        ElementTransformUtils.MirrorElements(
            doc,
            idsToMove,
            mirrorPlane,
            true);   // true = keep original, false = move original

    tx.Commit();
}
🟢public static void RotateElement(Document doc, ElementId elementId, Line axis, double angle)
Rotates a single element around the given axis line by angle (radians). The axis is a line in 3D space — both its direction and position matter. Use XYZ.BasisZ through the element's centre for standard plan-rotation.
🟢public static ICollection<ElementId> CopyElement(Document doc, ElementId elementId, XYZ translation)
Copies a single element with the given offset. Returns the ID of the new copy. The original is unchanged. All parameters, hosted elements, and associations are copied.
Returns: ICollection<ElementId> — IDs of the newly created copies (usually 1 element).
// Example — Copy elements with offsets (array pattern)
// ── Copy one wall offset by 5m ──
using (var tx = new Transaction(doc, "Copy Wall"))
{
    tx.Start();
    double offsetFt = UnitUtils.ConvertToInternalUnits(5000, UnitTypeId.Millimeters);
    ICollection<ElementId> newIds =
        ElementTransformUtils.CopyElement(doc, wall.Id, new XYZ(0, offsetFt, 0));
    Wall newWall = doc.GetElement(newIds.First()) as Wall;
    tx.Commit();
}

// ── Array: copy a column 5 times at equal spacing ──
using (var tx = new Transaction(doc, "Column Array"))
{
    tx.Start();
    double spacingFt = UnitUtils.ConvertToInternalUnits(6000, UnitTypeId.Millimeters);
    for (int i = 1; i <= 5; i++)
    {
        ElementTransformUtils.CopyElement(
            doc, column.Id, new XYZ(i * spacingFt, 0, 0));
    }
    tx.Commit();
}
🟢public static ICollection<ElementId> CopyElements(Document document, ICollection<ElementId> elementIds, XYZ translation)
Copies multiple elements with a translation offset. Returns IDs of all newly created copies in the same order as input. The most common copy overload for batch operations.
🟢public static ICollection<ElementId> CopyElements(Document sourceDocument, ICollection<ElementId> elementIds, Document destinationDocument, Transform transform, CopyPasteOptions options)
Copies elements from one document to another. This is the most powerful overload — it transfers element types, materials, nested families, and all associated data. The transform positions the copied elements in the destination document's coordinate space. Pass Transform.Identity to preserve original positions.
ParameterTypeDescription
sourceDocumentDocumentThe document to copy from.
elementIdsICollection<ElementId>IDs of elements to copy from the source.
destinationDocumentDocumentThe document to paste into (must be open).
transformTransformTransform applied to copied elements. Use Transform.Identity to keep original positions.
optionsCopyPasteOptionsControls duplicate type handling. Can be null.
Returns: ICollection<ElementId> — IDs of newly created elements in the destination document.
// Example — Cross-document copy: transfer walls + rooms from template
Application app = commandData.Application.Application;

// Open source (template) document
Document sourceDoc = app.OpenDocumentFile(@"C:\Templates\CoreLayout.rvt");
Document destDoc   = commandData.Application.ActiveUIDocument.Document;

// Get elements to copy from template
var toCopy = new FilteredElementCollector(sourceDoc)
    .OfClass(typeof(Wall))
    .WhereElementIsNotElementType()
    .ToElementIds()
    .ToList();

// CopyPasteOptions — control how duplicate types are handled
var cpOpts = new CopyPasteOptions();
cpOpts.SetDuplicateTypeNamesHandler(new DuplicateTypeNamesHandler());

using (var tx = new Transaction(destDoc, "Import Template Walls"))
{
    tx.Start();

    ICollection<ElementId> newIds =
        ElementTransformUtils.CopyElements(
            sourceDoc,
            toCopy,
            destDoc,
            Transform.Identity,   // preserve original positions
            cpOpts);

    System.Diagnostics.Debug.WriteLine(
        $"Copied {newIds.Count} walls into destination");

    tx.Commit();
}

// Close the source template
sourceDoc.Close(false);

// ── DuplicateTypeNamesHandler for cross-doc copy ──
public class DuplicateTypeNamesHandler : IDuplicateTypeNamesHandler
{
    public DuplicateTypeAction OnDuplicateTypeNamesFound(
        DuplicateTypeNamesHandlerArgs args)
    {
        // Options:
        // UseDestinationTypes — keep existing types in dest (ignore source)
        // UseSourceTypes      — overwrite dest types with source types
        
        return DuplicateTypeAction.UseDestinationTypes;
    }
}

GeometryCreationUtilities

Create Solid geometry objects from scratch — extrusions, sweeps, revolves, blends — without a family document. Used for DirectShape geometry, clash volumes, and custom analysis shapes.

ClassGeometryCreationUtilities
Inherits: Static utility
Namespace: Autodesk.Revit.DB · Assembly: RevitAPI.dll
Static factory class for creating Solid geometry objects programmatically without a family document. The resulting Solids can be used with DirectShape.SetShape(), BooleanOperationsUtils, or for geometric analysis. All methods take profile CurveLoops and return a Solid.
🟢 All Factory Methods
🟢public static Solid CreateExtrusionGeometry(IList<CurveLoop> profileLoops, XYZ extrusionDir, double extrusionDist)
Creates a solid extrusion from a list of CurveLoops. profileLoops[0] = outer boundary, subsequent loops = holes. extrusionDir is the direction of extrusion (must be perpendicular to profile plane). extrusionDist is the length in feet.
ParameterTypeDescription
profileLoopsIList<CurveLoop>Profile boundary loops. [0]=outer, [1..n]=holes.
extrusionDirXYZDirection to extrude (unit vector, perpendicular to profile).
extrusionDistdoubleExtrusion depth in feet. Must be positive.
Returns: Solid — The extruded solid geometry object.
// Example — Create L-shaped solid + use with DirectShape
// ── Create an L-shaped column cross-section ──
double w1 = UnitUtils.ConvertToInternalUnits(400, UnitTypeId.Millimeters);
double w2 = UnitUtils.ConvertToInternalUnits(200, UnitTypeId.Millimeters);
double h  = UnitUtils.ConvertToInternalUnits(3000, UnitTypeId.Millimeters);

// L-shape profile (6 points)
XYZ[] pts = {
    new XYZ(0,  0,  0),
    new XYZ(w1, 0,  0),
    new XYZ(w1, w2, 0),
    new XYZ(w2, w2, 0),
    new XYZ(w2, w1, 0),
    new XYZ(0,  w1, 0)
};
CurveLoop profile = new CurveLoop();
for (int i = 0; i < pts.Length; i++)
    profile.Append(Line.CreateBound(pts[i], pts[(i + 1) % pts.Length]));

// Create the solid (extrude upward)
Solid lColumnSolid = GeometryCreationUtilities.CreateExtrusionGeometry(
    new List<CurveLoop> { profile },
    XYZ.BasisZ,    // extrude upward
    h);

// Use it as a DirectShape
using (var tx = new Transaction(doc, "L-Column"))
{
    tx.Start();
    DirectShape ds = DirectShape.CreateElement(doc,
        new ElementId(BuiltInCategory.OST_StructuralColumns));
    ds.SetShape(new GeometryObject[] { lColumnSolid });
    ds.Name = "L-Column 400×400";
    tx.Commit();
}
🟢public static Solid CreateRevolvedGeometry(Frame coordinateFrame, IList<CurveLoop> profileLoops, double startAngle, double endAngle)
Creates a solid by revolving a profile around the Z-axis of the given coordinate frame. The profile must be on one side of the revolution axis. startAngle=0, endAngle=2π creates a full revolution (cylinder, cone, torus etc.).
// Example — Create a cylinder and a pipe-shaped solid
// ── Cylinder: revolve a rectangle ──
double radius = UnitUtils.ConvertToInternalUnits(300, UnitTypeId.Millimeters);
double cylH   = UnitUtils.ConvertToInternalUnits(1000, UnitTypeId.Millimeters);

// Profile: rectangle from (radius,0,0) to (radius,0,cylH)
// on the XZ plane — revolution axis is Z
CurveLoop cylProfile = new CurveLoop();
cylProfile.Append(Line.CreateBound(new XYZ(radius, 0, 0),   new XYZ(radius, 0, cylH)));
cylProfile.Append(Line.CreateBound(new XYZ(radius, 0, cylH), new XYZ(0, 0, cylH)));
cylProfile.Append(Line.CreateBound(new XYZ(0, 0, cylH),      new XYZ(0, 0, 0)));
cylProfile.Append(Line.CreateBound(new XYZ(0, 0, 0),        new XYZ(radius, 0, 0)));

// Frame: origin at base, Z = revolution axis
var frame = new Frame(XYZ.Zero, XYZ.BasisX, XYZ.BasisY, XYZ.BasisZ);

Solid cylinder = GeometryCreationUtilities.CreateRevolvedGeometry(
    frame,
    new List<CurveLoop> { cylProfile },
    0,
    2 * Math.PI);   // full revolution

System.Diagnostics.Debug.WriteLine(
    $"Cylinder volume: {cylinder.Volume * 0.0283168:F4}m³");
🟢public static Solid CreateSweptGeometry(Curve sweepPath, int pathAttachmentCurveIndex, double pathAttachmentParameter, IList<CurveLoop> profileLoops)
Creates a solid by sweeping a profile along a path curve. The profile is placed perpendicular to the path at the attachment point. Used for pipes, beams, rails, and any linear solid following a curve.
// Example — Swept rectangular section along an arc path
// Path: semicircle arc
Arc sweepPath = Arc.Create(XYZ.Zero, 5.0, 0, Math.PI,
    XYZ.BasisX, XYZ.BasisZ);

// Profile: 200×400mm rectangle centred at origin
double pw = UnitUtils.ConvertToInternalUnits(200, UnitTypeId.Millimeters) / 2;
double ph = UnitUtils.ConvertToInternalUnits(400, UnitTypeId.Millimeters) / 2;
CurveLoop profile = new CurveLoop();
profile.Append(Line.CreateBound(new XYZ(-pw,-ph,0),new XYZ( pw,-ph,0)));
profile.Append(Line.CreateBound(new XYZ( pw,-ph,0),new XYZ( pw, ph,0)));
profile.Append(Line.CreateBound(new XYZ( pw, ph,0),new XYZ(-pw, ph,0)));
profile.Append(Line.CreateBound(new XYZ(-pw, ph,0),new XYZ(-pw,-ph,0)));

Solid sweptBeam = GeometryCreationUtilities.CreateSweptGeometry(
    sweepPath,
    0,      // path attachment curve index (0 = the path itself)
    0.0,    // attachment parameter (0.0 = start of path)
    new List<CurveLoop> { profile });

System.Diagnostics.Debug.WriteLine(
    $"Swept beam volume: {sweptBeam.Volume * 0.0283168:F4}m³");
🟢public static Solid CreateBlendGeometry(CurveLoop topProfile, CurveLoop bottomProfile, IList<VertexPair> vertexPairing)
Creates a solid by blending (lofting) between two CurveLoops. The vertexPairing specifies which vertex in the top profile corresponds to which vertex in the bottom profile — controls the twist of the blend.
// Example — Blend from square base to circular top (tapered column)
double baseSize = UnitUtils.ConvertToInternalUnits(500, UnitTypeId.Millimeters);
double topRad   = UnitUtils.ConvertToInternalUnits(200, UnitTypeId.Millimeters);
double colH     = UnitUtils.ConvertToInternalUnits(3000, UnitTypeId.Millimeters);

// Bottom: square
double h = baseSize / 2;
CurveLoop bottom = new CurveLoop();
bottom.Append(Line.CreateBound(new XYZ(-h,-h,0),new XYZ( h,-h,0)));
bottom.Append(Line.CreateBound(new XYZ( h,-h,0),new XYZ( h, h,0)));
bottom.Append(Line.CreateBound(new XYZ( h, h,0),new XYZ(-h, h,0)));
bottom.Append(Line.CreateBound(new XYZ(-h, h,0),new XYZ(-h,-h,0)));

// Top: circle (approximate with arc)
Plane     topPlane  = Plane.CreateByNormalAndOrigin(XYZ.BasisZ, new XYZ(0,0,colH));
CurveLoop top       = new CurveLoop();
top.Append(Arc.Create(topPlane, topRad, 0, Math.PI));
top.Append(Arc.Create(topPlane, topRad, Math.PI, 2 * Math.PI));

Solid taperedCol = GeometryCreationUtilities.CreateBlendGeometry(
    top, bottom, null);  // null = auto vertex pairing

System.Diagnostics.Debug.WriteLine(
    $"Tapered column volume: {taperedCol.Volume * 0.0283168:F4}m³");

Shared Parameters + ParameterBinding

The complete workflow for creating shared parameters programmatically and binding them to categories — the #1 enterprise add-in requirement.

// Two Types of Programmatic Parameters
Shared Parameters — defined in an external .txt file, identified by GUID, transferable between projects, appear in schedules. Use for data that needs to travel with elements across projects.
Project Parameters — defined in the project only, not in a shared file. Created directly without a .txt file. Cannot be scheduled across projects.
ClassDefinitionFile (Shared Parameter File)
Namespace: Autodesk.Revit.DB · Accessed via app.OpenSharedParameterFile()
Represents the shared parameter .txt file. The file is plain text — Revit creates and manages it. Each file contains groups of parameter definitions, each with a unique GUID. You must set app.SharedParametersFilename before calling OpenSharedParameterFile().
🟢 Complete Shared Parameter Workflow
// Example — Complete shared parameter creation + binding workflow
Application app    = commandData.Application.Application;
Document    doc    = commandData.Application.ActiveUIDocument.Document;
string      spPath = @"C:\BIM\StudioSharedParams.txt";

// ── STEP 1: Create or open the shared parameter file ──
if (!System.IO.File.Exists(spPath))
    System.IO.File.Create(spPath).Close();  // create empty file

app.SharedParametersFilename = spPath;
DefinitionFile defFile = app.OpenSharedParameterFile();

// ── STEP 2: Get or create a definition group ──
const string GROUP_NAME = "Studio BIM Data";
DefinitionGroup group = defFile.Groups.get_Item(GROUP_NAME)
    ?? defFile.Groups.Create(GROUP_NAME);

// ── STEP 3: Create parameter definitions ──
var paramDefs = new[]
{
    // (name, spec type, is visible)
    ("FireRating",     SpecTypeId.String.Text,             true),
    ("UValue",         SpecTypeId.ThermalTransmittance,    true),
    ("CarbonKgM2",     SpecTypeId.MassPerUnitArea,         true),
    ("ReviewStatus",   SpecTypeId.String.Text,             true),
    ("IsExternal",     SpecTypeId.Boolean.YesNo,           true),
};

var createdDefs = new Dictionary<string, ExternalDefinition>();
foreach (var (name, spec, visible) in paramDefs)
{
    Definition existing = group.Definitions.get_Item(name);
    if (existing is ExternalDefinition ext)
    {
        createdDefs[name] = ext;  // already exists
    }
    else
    {
        var opts = new ExternalDefinitionCreationOptions(name, spec)
        {
            Visible = visible,
            UserModifiable = true
        };
        createdDefs[name] = group.Definitions.Create(opts) as ExternalDefinition;
    }
}

// ── STEP 4: Bind parameters to categories ──
CategorySet wallCats = app.Create.NewCategorySet();
wallCats.Insert(doc.Settings.Categories.get_Item(BuiltInCategory.OST_Walls));
wallCats.Insert(doc.Settings.Categories.get_Item(BuiltInCategory.OST_Floors));
wallCats.Insert(doc.Settings.Categories.get_Item(BuiltInCategory.OST_Roofs));

using (var tx = new Transaction(doc, "Bind Shared Params"))
{
    tx.Start();

    foreach (var kvp in createdDefs)
    {
        ExternalDefinition def = kvp.Value;
        if (def == null) continue;

        // Check if already bound
        BindingMap bm = doc.ParameterBindings;
        if (bm.Contains(def)) continue;  // skip if already bound

        // Create instance binding (appears on instances, not types)
        InstanceBinding binding = app.Create.NewInstanceBinding(wallCats);

        // OR: TypeBinding (appears on types/families)
        // TypeBinding typeBinding = app.Create.NewTypeBinding(wallCats);

        bool ok = doc.ParameterBindings.Insert(def, binding,
            GroupTypeId.Data);  // which group in Properties panel

        System.Diagnostics.Debug.WriteLine(
            $"Bound '{kvp.Key}': {(ok ? "OK" : "FAILED")}");
    }

    tx.Commit();
}

// ── STEP 5: Write values to elements ──
using (var tx = new Transaction(doc, "Set Shared Params"))
{
    tx.Start();
    foreach (var wall in walls)
    {
        // Access by GUID (most reliable)
        ExternalDefinition frDef = createdDefs["FireRating"];
        Parameter frParam = wall.get_Parameter(frDef.GUID);
        frParam?.Set("2HR");

        Parameter uvParam = wall.get_Parameter(createdDefs["UValue"].GUID);
        uvParam?.Set(0.35);  // W/m²K — Revit stores as internal units

        Parameter extParam = wall.get_Parameter(createdDefs["IsExternal"].GUID);
        extParam?.Set(1);  // 1 = Yes, 0 = No for YesNo params
    }
    tx.Commit();
}

// ── STEP 6: Read all shared parameters on an element ──
foreach (Parameter p in wall.Parameters)
{
    if (!p.IsShared) continue;
    string val = p.StorageType switch
    {
        StorageType.String    => p.AsString(),
        StorageType.Double   => (p.AsDouble() * 304.8).ToString("F2"),
        StorageType.Integer  => p.AsInteger().ToString(),
        StorageType.ElementId => p.AsElementId()?.IntegerValue.ToString(),
        _                    => "—"
    };
    System.Diagnostics.Debug.WriteLine(
        $"  [{p.Definition.Name}] = {val}");
}

GlobalParameter

Project-wide parameters that can drive multiple dimensions, element parameters, and constraints from a single value.

ClassGlobalParameter
Inherits: Element
Namespace: Autodesk.Revit.DB
A project-level parameter that can control dimensions and constrain element parameters. Global parameters appear in the Manage → Global Parameters dialog. They can drive wall heights, grid spacing, facade module widths, and any dimension or constrained parameter — making the model fully parametric.
🟢 Complete GlobalParameter Workflow
// Example — Create, read, write, and link global parameters
// ── Create global parameters ──
using (var tx = new Transaction(doc, "Global Params"))
{
    tx.Start();

    // Create a length global parameter
    GlobalParameter floorHeight = GlobalParameter.Create(
        doc,
        "FloorToCeilingHeight",
        SpecTypeId.Length);   // or SpecTypeId.String, SpecTypeId.Boolean.YesNo etc.

    // Set the value (in internal units)
    floorHeight.SetValue(
        new DoubleParameterValue(
            UnitUtils.ConvertToInternalUnits(3000, UnitTypeId.Millimeters)));

    // Create a text global parameter
    GlobalParameter projectCode = GlobalParameter.Create(
        doc, "ProjectCode", SpecTypeId.String.Text);
    projectCode.SetValue(new StringParameterValue("2024-BIM-001"));

    // Create a boolean
    GlobalParameter isMetric = GlobalParameter.Create(
        doc, "UseMetric", SpecTypeId.Boolean.YesNo);
    isMetric.SetValue(new IntegerParameterValue(1));  // 1=Yes

    tx.Commit();
}

// ── Read all global parameters ──
var globalParamIds = GlobalParametersManager.GetAllGlobalParameters(doc);
foreach (var gpId in globalParamIds)
{
    GlobalParameter gp = doc.GetElement(gpId) as GlobalParameter;
    if (gp == null) continue;

    ParameterValue val    = gp.GetValue();
    string         valStr = "?";

    if (val is DoubleParameterValue dpv)
        valStr = $"{dpv.Value * 304.8:F0}mm";
    else if (val is StringParameterValue spv)
        valStr = spv.Value;
    else if (val is IntegerParameterValue ipv)
        valStr = ipv.Value == 1 ? "Yes" : "No";

    System.Diagnostics.Debug.WriteLine(
        $"GlobalParam: {gp.Name} = {valStr} | Formula: {gp.GetFormula() ?? "(none)"}");
}

// ── Update a global parameter value ──
GlobalParameter fh = GlobalParametersManager
    .FindByName(doc, "FloorToCeilingHeight");

if (fh != null)
{
    using (var tx = new Transaction(doc, "Update GlobalParam"))
    {
        tx.Start();
        fh.SetValue(new DoubleParameterValue(
            UnitUtils.ConvertToInternalUnits(3200, UnitTypeId.Millimeters)));
        tx.Commit();
    }
}

// ── Set formula on a global parameter ──
using (var tx = new Transaction(doc, "GP Formula"))
{
    tx.Start();
    // Formula uses other global parameter names
    fh?.SetFormula("FloorHeight * 0.85");
    tx.Commit();
}

// ── Check what dimensions/params are driven by a global param ──
if (fh != null)
{
    var users = GlobalParametersManager.GetUsedGlobalParameters(doc, fh.Id);
    System.Diagnostics.Debug.WriteLine(
        $"GlobalParam '{fh.Name}' drives {users.Count} elements/dimensions");
}

Phase · PhaseFilter · Phasing Workflow

Create and manage construction phases, phase filters, and phase-aware element queries.

ClassPhase
Inherits: Element
Namespace: Autodesk.Revit.DB
Represents a construction phase in the Revit project. All model elements have a Phase Created and optionally a Phase Demolished parameter. Phases are stored in doc.Phases (a PhaseArray) and are always ordered chronologically.
🟢 Complete Phasing Workflow
// Example — Create phases, assign elements, create phase filter, phase-aware query
// ── Read existing phases ──
PhaseArray phases = doc.Phases;
System.Diagnostics.Debug.WriteLine($"Project has {phases.Size} phases:");
foreach (Phase ph in phases)
    System.Diagnostics.Debug.WriteLine($"  {ph.Id}: {ph.Name}");

Phase existingPhase      = phases.get_Item(0);   // first phase
Phase newConstructPhase  = phases.get_Item(phases.Size - 1); // last phase

// ── Create a new phase ──
using (var tx = new Transaction(doc, "Create Phase"))

    tx.Start();

    Phase phase2 = doc.Phases.get_Item(0);  // insert after phase index 0

    // Rename existing phase
    existingPhase.Name = "Phase 1 - Existing";

using (var tx = new Transaction(doc, "Assign Phases"))
{
    tx.Start();
    Phase extension = doc.Phases.get_Item(doc.Phases.Size - 1);

    foreach (var wall in newWalls)
    {
        // Set phase created
        wall.get_Parameter(BuiltInParameter.PHASE_CREATED)
            .Set(extension.Id);

        // Demolish existing walls in the extension zone
        // wall.get_Parameter(BuiltInParameter.PHASE_DEMOLISHED).Set(extension.Id);
    }
    tx.Commit();

Phase queryPhase = doc.Phases.get_Item(1);
var phaseFilter  = new ElementPhaseStatusFilter(queryPhase.Id,
    ElementPhaseStatus.Existing);

var existingWalls = new FilteredElementCollector(doc)
    .OfClass(typeof(Wall))
    .WherePasses(phaseFilter)
    .ToList();

// ElementPhaseStatus values:
// New          — created in this phase
// Existing     — existed before this phase and not demolished
// Demolished   — demolished in this phase
// Temporary    — created and demolished in same phase
// None         — not part of this phase

using (var tx = new Transaction(doc, "Set View Phase"))
{
    tx.Start();
    ViewPlan vp = ...;

    vp.get_Parameter(BuiltInParameter.VIEW_PHASE)
        .Set(queryPhase.Id);

// Autodesk.Revit.DB.Architecture

StairsEditScope · StairsRun · StairsLanding

The correct pattern for creating and modifying stairs — requires StairsEditScope, not a regular Transaction.

// Critical — Stairs require StairsEditScopeUnlike most elements, stairs CANNOT be created or modified inside a regular Transaction. You must use StairsEditScope — a special scope that wraps stair operations. Forgetting this causes an InvalidOperationException.
ClassStairsEditScope
Inherits: IDisposable
Namespace: Autodesk.Revit.DB.Architecture
A special scope required for all stair creation and modification. Works alongside a Transaction — the Transaction records the operation in the undo stack, while StairsEditScope manages the stair-specific editing state. Both must be started before making stair changes.
🟢 Complete Stair Creation + Modification
// Example — Create straight stair + add landing + modify
using Autodesk.Revit.DB.Architecture;

Level baseLvl = ...;   // ground floor
Level topLvl  = ...;   // first floor

Stairs stair;
ElementId stairId;

// ── STEP 1: Create stair using StairsEditScope + Transaction ──
using (var tx = new Transaction(doc, "Create Stairs"))
{
    tx.Start();

    using (var scope = new StairsEditScope(doc, "Create Stairs"))
    {
        stairId = scope.Start(baseLvl.Id, topLvl.Id);
        // Start() creates an empty stair assembly and returns its ID

        // Create a straight run
        StairsRun run = StairsRun.CreateStraightRun(
            doc,
            stairId,
            Line.CreateBound(
                new XYZ(0, 0, baseLvl.Elevation),
                new XYZ(0, 12, baseLvl.Elevation)),  // path of run
            StairsRunJustification.Center);

        // Configure run properties
        run.ActualRunWidth = UnitUtils.ConvertToInternalUnits(
            1200, UnitTypeId.Millimeters);

        // Commit the scope (validates stair geometry)
        scope.Commit(new StairsEditScopeFailuresPreprocessor());
    }

    tx.Commit();
}

// ── STEP 2: Modify existing stair ──
stair = doc.GetElement(stairId) as Stairs;

using (var tx = new Transaction(doc, "Modify Stair"))
{
    tx.Start();
    using (var scope = new StairsEditScope(doc, "Modify"))
    {
        scope.Start(stairId);  // pass existing stair ID to edit it

        // Read stair properties
        int    risers   = stair.DesiredRisersNumber;
        double riserH   = stair.ActualRiserHeight;

        // Modify desired riser count
        stair.DesiredRisersNumber = 18;

        // Access runs and landings
        foreach (ElementId rid in stair.GetStairsRuns())
        {
            StairsRun r = doc.GetElement(rid) as StairsRun;
            System.Diagnostics.Debug.WriteLine(
                $"  Run: risers={r.ActualRisersNumber} width={r.ActualRunWidth * 304.8:F0}mm");
        }

        foreach (ElementId lid in stair.GetStairsLandings())
        {
            StairsLanding land = doc.GetElement(lid) as StairsLanding;
            System.Diagnostics.Debug.WriteLine(
                $"  Landing area: {land.get_BoundingBox(null)?.Max.X:F2}");
        }

        scope.Commit(new StairsEditScopeFailuresPreprocessor());
    }
    tx.Commit();
}

// ── StairsEditScopeFailuresPreprocessor (suppress common warnings) ──
public class StairsEditScopeFailuresPreprocessor : IFailuresPreprocessor
{
    public FailureProcessingResult PreprocessFailures(FailuresAccessor acc)
    {
        foreach (var f in acc.GetFailureMessages())
            if (f.GetSeverity() == FailureSeverity.Warning)
                acc.DeleteWarning(f);
        return FailureProcessingResult.Continue;
    }
}

PathReinforcement · AreaReinforcement · BeamSystem

Structural reinforcement and beam system automation.

ClassPathReinforcement
Inherits: Element
Namespace: Autodesk.Revit.DB.Structure
Represents a linear reinforcement (rebar distributed along a path/curve) within a structural host element (floor, wall, or foundation). Unlike individual Rebar elements, PathReinforcement creates evenly-spaced bars along a curve with configurable spacing and cover.
🟢 Creation + AreaReinforcement + BeamSystem
// Example — PathReinforcement, AreaReinforcement, BeamSystem
using Autodesk.Revit.DB.Structure;

// ═══════════════════════════════════
// PATH REINFORCEMENT
// ═══════════════════════════════════
Floor        slab    = ...;
RebarBarType barType = new FilteredElementCollector(doc)
    .OfClass(typeof(RebarBarType)).Cast<RebarBarType>().First();

using (var tx = new Transaction(doc, "PathReinforcement"))
{
    tx.Start();

    // Define the path curve (along the slab centreline)
    var pathCurves = new CurveArray();
    pathCurves.Append(Line.CreateBound(
        new XYZ(0,0,0), new XYZ(10,0,0)));

    PathReinforcement pathReinf =
        PathReinforcement.Create(
            doc,
            slab,           // host
            pathCurves,     // path
            true,           // flipYDir
            barType.Id,     // primary bar type
            ElementId.InvalidElementId,  // no hook type start
            ElementId.InvalidElementId); // no hook type end

    // Set spacing
    pathReinf.get_Parameter(BuiltInParameter.REBAR_ELEM_SPACING_VALUE)
        ?.Set(UnitUtils.ConvertToInternalUnits(200, UnitTypeId.Millimeters));

    tx.Commit();
}

// ═══════════════════════════════════
// AREA REINFORCEMENT
// ═══════════════════════════════════
using (var tx = new Transaction(doc, "AreaReinforcement"))
{
    tx.Start();

    // Define boundary of reinforcement area
    var boundary = new CurveArray();
    boundary.Append(Line.CreateBound(new XYZ(0,0,0), new XYZ(10,0,0)));
    boundary.Append(Line.CreateBound(new XYZ(10,0,0), new XYZ(10,8,0)));
    boundary.Append(Line.CreateBound(new XYZ(10,8,0), new XYZ(0,8,0)));
    boundary.Append(Line.CreateBound(new XYZ(0,8,0), new XYZ(0,0,0)));

    AreaReinforcement areaReinf =
        AreaReinforcement.Create(
            doc,
            slab,
            boundary,
            XYZ.BasisX,        // major bar direction
            barType.Id,        // major bar type
            ElementId.InvalidElementId,
            barType.Id,        // minor bar type
            ElementId.InvalidElementId);

    tx.Commit();
}

// ═══════════════════════════════════
// BEAM SYSTEM
// ═══════════════════════════════════
FamilySymbol beamSym = new FilteredElementCollector(doc)
    .OfClass(typeof(FamilySymbol))
    .OfCategory(BuiltInCategory.OST_StructuralFraming)
    .Cast<FamilySymbol>().First();

using (var tx = new Transaction(doc, "Beam System"))
{
    tx.Start();
    if (!beamSym.IsActive) beamSym.Activate();

    // Create beam system from boundary curves
    var bsProfile = new CurveArray();
    bsProfile.Append(Line.CreateBound(new XYZ(0,0,3),  new XYZ(15,0,3)));
    bsProfile.Append(Line.CreateBound(new XYZ(15,0,3), new XYZ(15,10,3)));
    bsProfile.Append(Line.CreateBound(new XYZ(15,10,3),new XYZ(0,10,3)));
    bsProfile.Append(Line.CreateBound(new XYZ(0,10,3), new XYZ(0,0,3)));

    BeamSystem bs = BeamSystem.Create(doc, bsProfile, level, XYZ.BasisX);
    bs.BeamSystemType = beamSym.Id;

    // Set layout rule
    bs.ChangeTypeId(bs.GetTypeId());

    // Read beam count
    System.Diagnostics.Debug.WriteLine(
        $"Beam system created with {bs.GetBeamIds().Count} beams");

    tx.Commit();
}
